영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

부채널(side channel) 공격은 암호 알고리즘 자체의 수학적 약점이 아니라, 암호 장치가 동작할 때 부수적으로 새어 나오는 물리적 정보(전력 소비, 연산 시간, 전자파, 소음, 발열 등)를 관찰해 키를 추정하는 공격이다. 따라서 ①전력 분석, ③타이밍, ④전자기 분석은 모두 전형적인 부채널 공격이다.

②의 선택 암호문 공격(CCA)은 공격자가 임의의 암호문을 복호화시켜 얻은 결과를 이용해 키나 평문을 추론하는 수학적 암호분석(cryptanalysis) 기법으로, 물리적 부산물을 관찰하는 것이 아니다. 따라서 부채널 공격과 가장 거리가 먼 것은 ②번이다.

오답 분석

보기①

전력 분석(SPA·DPA)은 대표적 부채널 공격으로 전력 파형에서 키 비트를 추정한다.

보기②

정답(거리 멂). 선택 암호문 공격은 물리적 정보가 아닌 입·출력 관계를 이용하는 수학적 암호분석이다.

보기③

타이밍 공격은 연산 소요 시간 차이를 이용하는 부채널 공격이다.

보기④

전자기 분석은 장비에서 방출되는 전자파를 측정하는 부채널 공격이다.

관련 이론 정리

암호 공격은 크게 두 갈래다. 수학적 암호분석은 알고리즘의 구조적 약점을 노리며, 공격자가 가진 정보에 따라 암호문 단독(COA), 기지 평문(KPA), 선택 평문(CPA), 선택 암호문(CCA) 공격으로 구분된다. 반면 부채널 공격은 알고리즘이 수학적으로 안전해도 구현·물리 계층의 누출을 이용해 키를 얻는다. 대표적으로 단순 전력 분석(SPA)과 통계 기법을 쓰는 차분 전력 분석(DPA), 타이밍 공격(RSA·비교 연산의 시간 차), 전자기 분석, 음향·캐시 기반 공격이 있다. 대응책은 연산 시간·전력 소비를 입력과 무관하게 만드는 상수 시간 구현, 난수를 섞는 마스킹·블라인딩, 물리적 차폐, 노이즈 삽입이다. 스마트카드·HSM·IoT 기기처럼 공격자가 물리적으로 접근 가능한 환경에서 특히 중요하다. 감리에서는 암호 모듈 검증(KCMVP)과 물리적 보호 대책을 점검한다.

대응 상수 시간 구현, 마스킹, 차폐

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

서비스와 그 보안 강화 기술의 올바른 대응은 다음과 같다.

  • WWW(HTTP) → HTTPS(SSL/TLS 적용)
  • telnet → SSH(암호화 원격 접속)
  • 전자우편(SMTP) → PGP 또는 S/MIME
  • 전자결제 → SET(Secure Electronic Transaction)

③은 전자우편에 PGP를 대응시켜 정확하므로 정답은 ③번이다. ①은 WWW에 ssh를 붙여 잘못됐고(HTTPS여야 함), ②는 telnet에 https를 붙여 잘못됐으며(SSH여야 함), ④는 전자결제에 S/MIME(메일 보안)을 붙여 잘못됐다(SET이 적합).

오답 분석

보기①

보기②

보기③

관련 이론 정리

응용 계층별 보안 프로토콜은 서비스마다 별도로 발전했다. 웹은 HTTP에 SSL/TLS를 결합한 HTTPS로 기밀성·무결성·서버 인증을 확보한다. 원격 접속은 평문인 telnet·rlogin을 SSH가 대체해 암호화와 공개키 인증을 제공한다. 전자우편은 종단 간 암호화를 위해 PGP(신뢰의 웹, 사용자 간 상호 서명 방식)와 S/MIME(PKI·인증기관 계층 구조 기반)을 사용하며, 두 방식 모두 하이브리드 암호(본문은 대칭키, 세션키는 공개키)와 전자서명을 결합한다. 전자결제는 카드 정보 보호를 위한 SET, 결제 카드 산업 표준인 PCI-DSS가 관련된다. 감리에서는 서비스별로 평문 프로토콜이 남아 있지 않은지, 인증서·키 관리가 적절한지 점검한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

RSA의 안전성은 큰 합성수 N = p×q를 소인수분해하기 어렵다는 가정에 전적으로 의존한다. 만약 N을 고속으로 소인수분해할 수 있다면 p, q를 얻어 오일러 함수 φ(N) = (p−1)(q−1)를 계산할 수 있고, 공개된 E로부터 개인키 D를 구할 수 있어 RSA가 완전히 해독된다.

②는 "소인수분해를 고속으로 할 수 있더라도 RSA 해독이 불가능하다"고 했으므로 안전성 근거를 정면으로 부정하는 틀린 설명이다. 따라서 정답은 ②번이다. 참고로 양자컴퓨터의 쇼어(Shor) 알고리즘이 소인수분해를 다항 시간에 해결할 수 있어 RSA를 위협하며, 이것이 양자내성암호(PQC) 전환의 이유다.

①(서명 가능), ③(C = M^E mod N), ④(M = C^D mod N)는 모두 옳다.

오답 분석

보기①

관련 이론 정리

RSA는 두 큰 소수 p, q를 골라 N = p·q, φ(N) = (p−1)(q−1)를 구하고, φ(N)과 서로소인 E를 공개 지수로, E·D ≡ 1 (mod φ(N))을 만족하는 D를 개인 지수로 삼는다. 공개키는 (E, N), 개인키는 (D, N)이며 암호화 C = M^E mod N, 복호화 M = C^D mod N이다. 전자서명은 반대로 개인키로 서명하고 공개키로 검증한다. 안전성 확보를 위해 키 길이 2048비트 이상을 권장하고, 같은 소수 재사용 금지, 안전한 난수로 소수 생성, 패딩(OAEP·PSS) 적용이 필요하다(패딩 없는 교과서 RSA는 결정적이라 취약). 부채널 공격(타이밍·전력)에 대비한 블라인딩도 요구된다. 감리에서는 키 길이·패딩 방식·키 보관을 점검한다.

정답 해설

IPSec의 두 동작 모드는 다음과 같다.

  • 전송(Transport) 모드: 원래 IP 헤더는 그대로 두고 IP 페이로드(상위 계층 데이터)만 암호화한다. 종단 호스트 간 통신에 사용된다.
  • 터널(Tunnel) 모드: 원본 IP 헤더를 포함한 IP 패킷 전체를 암호화한 뒤 새로운 IP 헤더를 붙여 캡슐화한다. 게이트웨이 간(사이트 간) VPN에 사용되며 내부 주소를 숨길 수 있다.

①은 이 둘을 정확히 뒤바꿔 서술했으므로(터널=페이로드만, 전송=전체) 틀렸다. 따라서 정답은 ①번이다. ②(SSL VPN의 응용 계층 보호), ③(L2TP는 PPTP+L2F 통합), ④(L2F는 Cisco 개발, 데이터링크 계층 캡슐화)는 모두 옳다.

보기②

관련 이론 정리

IPSec은 3계층에서 보안을 제공하는 프로토콜 모음으로, AH(Authentication Header)는 무결성·인증·재전송 방지를 제공하지만 기밀성(암호화)은 제공하지 않고, ESP(Encapsulating Security Payload)는 여기에 기밀성까지 제공한다. 키 교환과 보안 연관(SA) 설정은 IKE가 담당한다. 모드는 앞서 정리한 전송·터널 모드로 나뉘며, NAT 환경에서는 AH가 무결성 검사에 IP 헤더를 포함해 통과가 어려워 ESP + NAT-T(UDP 4500 캡슐화)를 사용한다. 사이트 간 VPN은 게이트웨이가 대신 캡슐화하므로 터널 모드가, 호스트 간 직접 보호는 전송 모드가 적합하다. SSL/TLS VPN은 응용 계층에 가까워 브라우저만으로 접속 가능해 원격 사용자 지원에 유리하다. 감리에서는 VPN 모드·알고리즘·인증 방식의 적정성을 점검한다.

모드 암호화 범위 용도

터널 모드 IP 패킷 전체 + 새 헤더 게이트웨이 간(사이트 간)
실무 적용 사례

VPN 구축 감리 시 사이트 간 연결에 터널 모드가 적용됐는지, NAT 구간에서 NAT-T 설정이 되어 있는지, 취약 암호 알고리즘(DES·MD5) 사용 여부를 점검한다.

감리사 시험 출제 포인트

전송 모드=페이로드만, 터널 모드=전체+새 헤더를 뒤바꾼 서술이 가장 전형적인 오답이다. AH(기밀성 없음)·ESP 구분도 함께 출제된다.

관련 이론 정리

비식별 조치는 식별자 제거 → 준식별자 처리 → 프라이버시 모델 적용 → 적정성 평가 → 사후관리의 절차로 이뤄진다. 처리 기법에는 가명처리(이름을 임의값으로), 총계처리(합계·평균으로 대체), 데이터 삭제, 범주화(나이 32세 → 30대), 데이터 마스킹(부분 은닉)이 있다. 이렇게 처리한 결과가 충분히 안전한지 판정하는 기준이 k-익명성 → ℓ-다양성 → t-근접성의 프라이버시 모델이며 뒤로 갈수록 강한 보호를 제공한다. 재식별 위험은 다른 데이터와의 결합(linkage)으로 커지므로, 비식별 정보라도 결합 가능성을 지속 평가해야 한다. 현행법상 완전한 익명정보는 개인정보가 아니지만 가명정보는 개인정보로 남아 안전조치 의무가 따른다. 감리에서는 비식별 조치의 적정성 평가와 재식별 모니터링을 점검한다.

t-근접성 쏠림·배경지식 공격
실무 적용 사례

데이터 개방·활용 사업 감리 시 비식별 조치 결과에 k-익명성 등 프라이버시 모델이 적용·검증됐는지, 적정성 평가단 운영과 재식별 모니터링이 이뤄지는지 점검한다.

정답 해설

파일시스템은 디스크를 클러스터(블록) 단위로 할당하므로, 파일의 실제 크기가 클러스터 경계와 딱 맞지 않으면 마지막 클러스터에 사용되지 않는 빈 공간이 남는다. 이 영역을 파일 슬랙(file slack)이라 하며, 이전에 그 자리에 있던 삭제 파일의 데이터가 그대로 남아 있을 수 있어 정보 유출 위험이자 동시에 디지털 포렌식의 중요한 증거 원천이 된다. 문제 설명과 정확히 일치하므로 정답은 ③번이다.

①저널은 파일시스템 변경 이력을 기록해 복구를 돕는 영역, ②메타데이터는 파일의 속성 정보, ④파일 지시자(디스크립터)는 프로세스가 열린 파일을 참조하는 정수 값으로 모두 다른 개념이다.

관련 이론 정리

디지털 포렌식에서 슬랙 공간은 삭제·은닉 데이터를 찾는 핵심 영역이다. 유형으로는 파일 끝과 섹터 끝 사이의 RAM 슬랙, 섹터 끝과 클러스터 끝 사이의 드라이브 슬랙(파일 슬랙), 파티션 미할당 영역인 볼륨 슬랙이 있다. 일반적인 삭제는 파일시스템의 링크만 제거하고 실제 데이터는 남기므로, 복구 도구로 되살릴 수 있다(그래서 포렌식에서 삭제 파일 복구가 가능하다). 따라서 중요 정보 폐기 시에는 완전 삭제(와이핑, 다중 덮어쓰기)나 물리적 파기·디가우징이 필요하며, SSD는 웨어 레벨링 때문에 덮어쓰기가 보장되지 않아 암호화 후 키 폐기(crypto-erase)가 권장된다. 포렌식 절차는 무결성 확보를 위해 쓰기 방지 장치 사용·해시값 산출·연계보관성(CoC) 기록을 지킨다. 감리에서는 매체 폐기 절차와 기록 관리를 점검한다.

정보자산 폐기 감리 시 저장매체 폐기 절차(와이핑·디가우징·물리 파기)와 폐기 확인서 기록, SSD의 암호화 기반 폐기 적용 여부를 점검한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

모델 역전 공격(Model Inversion Attack)은 공격자가 AI 모델에 다수의 질의를 보내고 그 출력(확률·신뢰도)을 분석해 학습에 사용된 원본 데이터나 그 특징을 복원하는 공격이다. 예컨대 얼굴 인식 모델의 출력을 반복 관찰해 학습에 쓰인 얼굴 이미지를 재구성할 수 있어 프라이버시 침해로 직결된다. 문제 설명과 정확히 일치하므로 정답은 ②번이다.

①Poisoning은 학습 단계에서 오염된 데이터를 주입해 모델 자체를 왜곡시키는 공격, ③Evasion은 추론 단계에서 입력에 미세한 교란(적대적 예제)을 가해 오분류를 유도하는 공격, ④Replay는 이전에 가로챈 유효 데이터를 재전송하는 전통적 네트워크 공격이다.

보기②

보기④

관련 이론 정리

AI 시스템 위협은 생명주기 단계별로 구분된다. 학습 단계에서는 오염 데이터를 넣는 포이즈닝과 특정 입력에만 오작동하도록 심는 백도어(트로이) 공격이 있다. 추론 단계에서는 미세 교란으로 오분류를 유도하는 회피(Evasion, 적대적 예제) 공격이 대표적이다. 프라이버시 침해 계열로는 학습 데이터를 복원하는 모델 역전, 특정 데이터가 학습에 쓰였는지 판별하는 멤버십 추론, 모델 자체를 복제하는 모델 추출(steal) 공격이 있다. 대응책은 차등 프라이버시(differential privacy) 적용, 출력 신뢰도 정보 제한(top-1만 반환), 질의 횟수·속도 제한, 적대적 훈련, 학습 데이터 검증, 모델 워터마킹이다. 감리에서는 AI 서비스의 API 남용 통제와 학습 데이터 보호 조치를 점검한다.

정답 해설

DRM은 문서 자체에 암호와 사용 권한 정책을 결합해, 파일이 외부로 유출되어도 인가된 환경에서만 열람·편집·인쇄·복사가 가능하도록 통제한다. 이때 권한은 사용자·그룹 단위뿐 아니라 문서를 여는 프로세스(애플리케이션) 단위로도 제어되어, 허가되지 않은 프로그램이 문서를 열거나 내용을 추출하는 것을 막고 화면 캡처 프로그램도 차단한다.

④는 "프로세스별로는 권한관리 기능을 제공하지 않는다"고 서술했으나 실제 E-DRM은 프로세스 단위 통제를 제공하므로 틀렸다. 따라서 정답은 ④번이다. ①②③은 모두 옳은 설명이다.

관련 이론 정리

DRM은 보호 대상에 따라 콘텐츠 DRM(음원·영상·전자책의 불법 복제 방지)과 E-DRM(기업 문서 보안)으로 나뉜다. E-DRM의 동작은 ① 문서 생성·저장 시 자동 암호화, ② 열람 요청 시 DRM 서버에서 사용자 권한 확인, ③ 허용 범위 내에서만 복호화·사용, ④ 사용 이력 로깅의 순서다. 통제 가능한 권한에는 열람·편집·인쇄·복사(클립보드)·화면 캡처·유효기간·오프라인 사용 횟수가 있고, 사용자·부서·기기·프로세스 단위로 세분화된다. 유사 솔루션인 DLP는 문서가 나가는 경로(메일·USB·웹)를 감시·차단하는 반면 DRM은 문서 자체를 보호해 유출 후에도 통제가 유지된다는 차이가 있다. 실무에서는 두 솔루션을 함께 운영한다. 감리에서는 DRM 적용 범위, 예외 문서 관리, 사용 이력 점검 체계를 확인한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

Burp Suite, Paros, WebScarab은 모두 브라우저와 웹 서버 사이에 위치해 HTTP 요청·응답을 가로채고 변조·재전송하며 취약점을 찾는 웹 프록시 기반 진단 도구다.

반면 ②Hydra는 SSH·FTP·HTTP 등 다양한 서비스에 대해 아이디·비밀번호를 무차별 대입(brute force)하는 패스워드 크래킹 도구로, 웹 취약점 진단 도구와는 목적이 다르다. 따라서 가장 거리가 먼 것은 ②번이다.

정답 해설

디지털 포렌식 5대 원칙에서 가장 명확한 판별 지점은 '마'다. 정당성의 원칙은 증거가 적법절차를 거쳐 수집되어야 한다는 것으로, 우리 형사소송법의 위법수집증거 배제 법칙에 따라 위법하게 수집된 증거는 수사관이 동행했더라도 증거능력이 인정되지 않는다. 따라서 '마'는 명백히 틀린 서술이고, 마를 포함한 ④는 제외된다.

'가'(신속성), '라'(연계보관성)는 정의가 정확하며, 원본 정답은 여기에 '나'·'다'까지 포함한 ③(가, 나, 다, 라)이다.

다만 학습 시 유의할 점은 통상적 정의로는 '나'와 '다'의 설명이 서로 바뀌어 있다는 것이다. 표준적으로 무결성의 원칙이 "수집된 증거가 위·변조되지 않았음을 증명"이고, 재현의 원칙이 "같은 조건에서 항상 같은 결과"에 해당한다. 시험에서는 이 두 원칙의 정의를 바꿔 제시하는 경우가 많으므로 명칭-정의 짝을 정확히 익혀 두어야 한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

Bell-LaPadula(BLP) 모델은 기밀성 보호를 목적으로 두 가지 핵심 규칙을 둔다.

  • 단순보안규칙(Simple Security Property): No Read Up — 주체는 자신보다 높은 등급의 객체를 읽을 수 없다.
  • *-보안규칙(Star Property): No Write Down — 주체는 자신보다 낮은 등급의 객체에 쓸 수 없다(상위 등급 정보가 하위로 흘러내리는 것을 방지).

문제는 Top Secret 취급자가 Secret 파일(하위 등급)에 쓰는 것을 금지하는 규칙을 묻고 있으므로 이는 No Write Down, 즉 *-보안규칙이다. 따라서 정답은 ②번이다.

③강한 *-규칙은 읽기·쓰기를 자신과 동일한 등급에서만 허용하는 더 엄격한 규칙이고, ④단순 무결성규칙은 Biba 모델(무결성)의 규칙으로 No Read Down에 해당한다.

관련 이론 정리

보안 모델은 보호 목표에 따라 규칙 방향이 반대다. BLP(기밀성)는 정보가 아래로 새는 것을 막으므로 No Read Up + No Write Down이고, Biba(무결성)는 낮은 신뢰 정보가 위로 올라가 오염시키는 것을 막으므로 No Read Down + No Write Up으로 정반대다. 이 두 쌍을 혼동하지 않는 것이 시험의 핵심이다. 그 밖에 Clark-Wilson은 상업 환경의 무결성을 위해 잘 형성된 트랜잭션과 직무 분리를 강조하고, Chinese Wall(Brewer-Nash)은 이해충돌 방지를 위해 경쟁사 정보에 동시 접근을 차단한다. 이런 모델은 MAC(강제적 접근통제)의 이론적 기반이며, 실무의 DAC(임의적 접근통제)·RBAC(역할 기반)과 함께 접근통제 정책을 구성한다. 감리에서는 등급 분류 체계와 접근통제 정책의 일관성을 점검한다.

보안 모델 규칙 비교:

모델 목표 읽기 쓰기
BLP 기밀성 No Read Up No Write Down
Biba 무결성 No Read Down No Write Up
실무 적용 사례

기밀 등급 체계를 운영하는 시스템 감리 시 등급별 접근 권한이 No Read Up·No Write Down 원칙에 맞게 구현됐는지, 하위 등급으로의 정보 유출 경로가 없는지 점검한다.

감리사 시험 출제 포인트

BLP: No Read Up(단순) / No Write Down(*-속성), Biba는 정반대를 세트로 암기한다. 등급 이동 방향을 묻는 형태가 반복된다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

ARP는 IP를 MAC으로 변환하는 데이터링크 계층(2계층) 프로토콜이며, ARP 기반 스니핑 공격은 MAC 주소 매핑을 위조해 트래픽을 공격자에게 흐르게 만든다.

③은 "네트워크 계층에서 통신하는 서버와 클라이언트의 IP 주소를 공격자의 IP 주소로 속인다"고 서술했는데, ARP 공격은 계층도 대상도 다르므로(2계층에서 MAC 주소를 속임) 틀린 설명이다. 따라서 정답은 ③번이다.

①(스위치 재밍/MAC 플러딩: 위조 MAC으로 주소 테이블을 넘치게 해 스위치를 허브처럼 동작시킴), ②(ARP Redirect: 공격자를 라우터로 믿게 함), ④(위조 ARP Request에 대한 응답 여부로 무차별 모드 호스트를 탐지)는 모두 옳다.

보기②

보기④

관련 이론 정리

스니핑은 네트워크를 오가는 데이터를 수동적으로 도청하는 공격이다. 허브 환경에서는 모든 프레임이 전 포트로 전달되므로 NIC를 무차별 모드(promiscuous mode)로 두기만 하면 도청이 가능했지만, 스위치 환경에서는 목적지 포트로만 전달되므로 공격자가 트래픽을 자신에게 끌어와야 한다. 그 방법이 ARP 스푸핑/Redirect(MAC 매핑 위조), 스위치 재밍(MAC 플러딩)(테이블 고갈로 플러딩 유도), ICMP Redirect(경로 변경 유도), 포트 미러링 악용이다. 탐지 기법으로는 위조 ARP Request에 대한 응답을 보는 방법, 존재하지 않는 MAC 주소로 ping을 보내 응답하는 호스트를 찾는 방법, DNS 역질의 관찰이 있다. 근본 대응은 종단 간 암호화(HTTPS·SSH·VPN)와 DAI·정적 ARP다. 감리에서는 내부망 평문 통신 여부와 스니핑 대응 설정을 점검한다.

ICMP Redirect 경로 변경 유도
대응 암호화, DAI, 정적 ARP
실무 적용 사례

내부망 감리 시 관리자 접속·중요 업무 구간에 평문 프로토콜이 쓰이는지, ARP 스푸핑 방지(DAI) 설정과 이상 ARP 트래픽 모니터링이 있는지 점검한다.

관련 이론 정리

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

「개인정보 보호법」 제3조 제1항은 "개인정보처리자는 개인정보의 처리 목적을 명확하게 하여야 하고 그 목적에 필요한 범위에서 최소한의 개인정보만을 적법하고 정당하게 수집하여야 한다"고 규정한다. 즉 목적 명확화는 최소 수집의 전제이며 생략할 수 없는 의무다.

②는 "처리 목적을 명확하게 하지 않아도"라고 하여 이 의무를 부정했으므로 틀렸다. 따라서 정답은 ②번이다.

①(제3조 제4항, 권리 침해 가능성·위험 정도를 고려한 안전 관리), ③(제3조 제6항, 책임·의무 준수로 신뢰 확보 노력), ④(제3조 제3항, 정확성·완전성·최신성 보장)는 모두 법이 정한 보호 원칙이다.

관련 이론 정리

「개인정보 보호법」 제3조는 OECD 8원칙을 국내법으로 수용한 조항으로, ① 목적 명확화 및 최소 수집, ② 목적 외 활용 금지(수집 목적 범위에서 적법하게 처리), ③ 정확성·완전성·최신성 보장, ④ 위험 정도를 고려한 안전 관리, ⑤ 처리방침 공개 등 투명성 확보와 정보주체 권리 보장, ⑥ 사생활 침해 최소화 방법으로 처리, ⑦ 가능한 경우 익명·가명 처리 우선, ⑧ 책임·의무 준수로 신뢰 확보를 규정한다. 특히 ⑦은 개정법에서 익명처리 우선, 곤란하면 가명처리의 순서를 명시해 데이터 활용 시 보호 수준의 우선순위를 정했다. 이 원칙들은 개별 조항 해석의 지침이 되며, 위반 시 시정명령·과징금의 판단 근거가 된다. 감리에서는 수집 항목의 최소성, 목적 외 이용 통제, 처리방침 공개를 점검한다.

실무 적용 사례

개인정보 처리 감리 시 수집 항목이 목적 달성에 꼭 필요한 최소 범위인지, 목적이 처리방침·동의서에 구체적으로 특정됐는지 점검한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

정보보호 위험관리에서 정보자산 식별은 관리체계 범위 내의 모든 정보자산을 누락 없이 식별한 뒤, 각 자산의 중요도(기밀성·무결성·가용성 관점)를 평가해 등급을 매기는 활동이다. 처음부터 "일부 중요 자산만" 식별하면 식별되지 않은 자산이 위험 평가에서 통째로 빠져 보호 사각지대가 생긴다.

③은 "일부 중요 정보자산만을 식별"한다고 서술했으므로 위험관리의 기본 원칙에 어긋난다. 따라서 정답은 ③번이다. ①(현황·흐름분석과 최신성 유지), ②(위협정보 수집과 위험평가 방법 선정), ④(보호대책 선정과 이행계획 수립)는 모두 옳다.

오답 분석

보기①

옳음. 관리체계 전 영역의 현황·흐름을 문서화하고 주기적으로 최신화한다.

보기②

옳음. 환경분석·위협정보 수집을 거쳐 적합한 위험평가 방법으로 위험을 평가한다.

보기③

정답(부적절). 범위 내 모든 정보자산을 식별해야 하며 일부만 식별하면 안 된다.

보기④

옳음. 위험 처리를 위한 보호대책을 선정하고 우선순위·일정·담당자·예산을 포함한 이행계획을 수립한다.

관련 이론 정리

ISMS-P 기준의 위험관리 절차는 ① 관리체계 범위 설정 → ② 현황 및 흐름분석 → ③ 정보자산 식별·중요도 산정 → ④ 위험 식별 및 평가 → ⑤ 보호대책 선정·이행계획 수립 → ⑥ 이행 및 효과 점검의 순환 구조다. 자산 식별은 하드웨어·소프트웨어·데이터·문서·인력·시설 등 범위 내 모든 자산을 목록화하고 담당자를 지정하는 것이며, 중요도는 CIA 관점의 평가로 산정한다. 위험 = 자산가치 × 위협 × 취약점의 개념으로 산정되며, 조직은 감내할 수 있는 위험 수용 수준(DoA)을 정해 이를 초과하는 위험에 대해 위험 감소·회피·전가(보험·아웃소싱)·수용 중 대응 전략을 선택한다. 감리에서는 자산 목록의 완전성, 위험평가 방법의 일관성, 이행계획의 실행 여부를 점검한다.

위험관리 절차:

단계 내용
현황·흐름분석 업무·정보 흐름 문서화, 최신성 유지
자산 식별 범위 내 전 자산 식별 + 중요도 산정
위험 평가 위협·취약점 분석, 위험도 산정
보호대책 선정 대책 선정 + 이행계획(우선순위·예산)
실무 적용 사례

ISMS 인증·정보보호 감리 시 자산 목록의 누락 여부(그림자 IT·클라우드 자산 포함), 위험평가 기준의 일관성, 이행계획의 실제 완료율을 점검한다.
