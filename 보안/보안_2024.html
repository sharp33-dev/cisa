영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

finger는 시스템에 등록된 사용자 계정명, 실명, 홈 디렉터리, 최종 로그인 시각 등을 조회하는 명령어·서비스다. 원격에서 finger @대상호스트 형태로 질의하면 유효한 계정 목록을 열거할 수 있어, 공격자가 무차별 대입이나 사회공학 공격 대상을 고르는 사전 정보 수집(reconnaissance) 단계에 악용된다. 따라서 정답은 ①번이다.

②echo는 문자열을 출력하는 기본 명령, ③NFS는 원격 파일시스템 공유 서비스(설정 미흡 시 파일 노출 위험은 있으나 사용자 정보 조회 목적이 아님), ④talk은 사용자 간 실시간 대화 유틸리티로 계정 정보 수집 도구가 아니다.

오답 분석

보기①

정답. finger는 등록 사용자 정보를 조회해 계정 열거에 악용된다.

보기②

echo는 인자로 받은 문자열을 출력하는 명령으로 사용자 정보 조회 기능이 없다.

보기③

NFS는 원격 파일 공유 서비스로 파일 접근과 관련되며 계정 정보 조회 용도가 아니다.

보기④

talk은 로그인한 사용자 간 대화 기능으로 정보 수집 목적의 도구가 아니다.

관련 이론 정리

공격은 일반적으로 정보 수집 → 취약점 분석 → 침투 → 권한 상승 → 흔적 제거의 단계를 거치며, 첫 단계인 정보 수집에서 대상의 계정·서비스·버전을 파악한다. 유닉스/리눅스에서 계정 정보와 관련된 요소로 /etc/passwd(계정·홈·셸, 암호는 /etc/shadow로 분리), finger(사용자 상세), rusers·rwho(원격 사용자 목록), who·w(현재 접속자)가 있다. finger·rlogin·rsh·telnet 같은 평문·정보 노출형 레거시 서비스는 비활성화하고 SSH로 대체하는 것이 보안 기준이다. 서버 보안 점검 항목에도 "불필요한 서비스 제거"가 포함된다. 감리에서는 불필요 서비스 구동 여부, 계정 정책, 로그인 시도 모니터링을 점검한다.

정보 수집 관련 명령/서비스:

항목 노출 정보
finger 계정명·실명·홈·최종 로그인
rusers/rwho 원격 접속 사용자 목록
who/w 현재 접속자
/etc/passwd 계정·셸·홈 디렉터리
실무 적용 사례

서버 보안 감리 시 finger·telnet·rsh 등 취약 서비스의 구동 여부를 점검하고, 비활성화 및 SSH 전환, 계정 열거 방지 설정을 권고한다.

감리사 시험 출제 포인트

finger는 사용자 정보 노출 → 계정 열거에 악용된다는 점이 핵심이다. 불필요 서비스 제거가 대응책으로 함께 출제된다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

제로 트러스트는 방화벽으로 내·외부를 나누고 내부는 신뢰하던 전통적 경계 기반(perimeter) 보안의 한계를 극복하기 위해 등장한 개념으로, "절대 신뢰하지 말고 항상 검증하라(Never trust, always verify)"를 원칙으로 한다. 내부에 있다는 이유만으로 신뢰하지 않고 접근 시마다 사용자·기기를 인증·인가하며 최소권한을 적용한다.

④는 "경계 기반 보안체계 구축을 더욱 강화해야 한다"고 서술했는데, 이는 제로 트러스트가 탈피하고자 하는 방향과 정반대이므로 틀린 설명이다. 따라서 정답은 ④번이다. ①(침해 가정), ②(지속 노출·잠재 침해 전제의 원칙 집합), ③(요건 미충족 시 접근 제한)은 모두 옳다.

오답 분석

보기①

옳음. 제로 트러스트는 네트워크가 이미 침해되었다고 가정(assume breach)한다.

보기②

옳음. 잠재적으로 침해될 수 있는 구성요소·사용자를 다루는 원칙 집합으로 정의된다.

보기③

옳음. 인증·인가 요건을 갖추지 못한 주체의 접근을 제한한다.

보기④

정답(부적절). 제로 트러스트는 경계 기반 보안에서 탈피하는 개념으로 서술이 정반대다.

관련 이론 정리

제로 트러스트(NIST SP 800-207)의 핵심 원칙은 ① 모든 데이터·컴퓨팅 서비스를 자원으로 간주, ② 네트워크 위치와 무관하게 모든 통신을 보호, ③ 세션 단위로 자원 접근 허용, ④ 동적 정책(사용자·기기 상태·행위 속성)에 따라 접근 결정, ⑤ 모든 자산의 무결성·보안 상태 지속 모니터링, ⑥ 접근 허용 전 인증·인가를 엄격히 수행, ⑦ 자산·트래픽 정보를 수집해 보안 개선에 활용이다. 구현 요소로는 정책 결정 지점(PDP)·정책 시행 지점(PEP), 강력한 IAM·다중 인증(MFA), 마이크로 세그멘테이션, 기기 상태 검증(NAC·EDR), 지속적 로그·행위 분석이 있다. 재택근무·클라우드 확산으로 경계가 흐려지면서 도입이 가속됐다. 감리에서는 접근통제 정책의 세분화와 지속 검증 체계를 점검한다.

전통 경계 보안 vs 제로 트러스트:

구분 경계 기반 제로 트러스트
신뢰 전제 내부는 신뢰 아무것도 신뢰 안 함
검증 시점 최초 진입 접근 시마다
통제 단위 네트워크 경계 자원·세션 단위
대표 기술 방화벽·VPN IAM·MFA·마이크로 세그멘테이션
실무 적용 사례

제로 트러스트 도입 감리 시 다중 인증 적용 범위, 기기 상태 검증, 마이크로 세그멘테이션 설계, 접근 로그의 지속 분석 체계를 점검한다.

감리사 시험 출제 포인트

제로 트러스트는 경계 기반 강화가 아니라 탈피라는 점이 핵심 함정이다. "Never trust, always verify"와 침해 가정 전제를 기억한다.

④ NAC : 네트워크 접근제어 솔루션으로 엔드 포인트 단말기가 회사의 내부 네트워크에 접속을 시도할 때 이를 제어하고 통제한다.

정답 해설

④의 NAC(Network Access Control) 설명은 정확하다. NAC는 엔드포인트 단말이 내부망 접속을 시도할 때 인증·보안 상태를 점검해 접근을 제어·통제하는 솔루션이다. 따라서 정답은 ④번이다.

①은 틀림: 메일 서버 앞단에서 스팸·폭탄 메일을 차단하는 것은 DNS가 아니라 메일 보안 게이트웨이(안티스팸 솔루션)다. DNS는 도메인-IP 변환 서비스다. ②③은 설명이 서로 뒤바뀌었다: 문서의 열람·편집·인쇄 권한을 통제하는 것은 DRM이고, 사용자 단의 다양한 전송 인터페이스(메일·USB·웹)를 통제해 기밀정보 유출을 막는 것은 DLP다.

정보보호 솔루션은 보호 대상과 통제 지점으로 구분된다. 영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

코드의 문제는 마지막 줄 printf(name);이다. printf의 첫 인자는 형식 문자열(format string)인데 여기에 사용자 입력을 그대로 전달하고 있다. 공격자가 이름에 %x %x %x나 %n을 입력하면 형식 지정자로 해석되어 스택 메모리 값이 노출되거나(%x, %s) 심하면 임의 메모리에 값을 기록(%n)해 코드 실행까지 가능해진다. 이것이 포맷 스트링 취약점이므로 정답은 ③번이다.

주의할 점은 입력을 받는 scanf("%99s", name)은 길이를 99자로 제한하고 있어 버퍼 오버플로우는 발생하지 않는다는 것이다(그래서 ①은 오답). 올바른 수정은 printf("%s", name);처럼 형식 문자열을 상수로 고정하고 사용자 입력은 인자로 전달하는 것이다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

GCM(Galois/Counter Mode)은 CTR 모드로 기밀성을 제공하면서 GMAC(갈루아 필드 기반 인증 태그)으로 무결성·인증까지 함께 제공하는 대표적인 인증 암호(AEAD, Authenticated Encryption with Associated Data) 모드다.

④는 "기밀성은 제공하지만 무결성은 보장하지 않는다"고 서술했으나, GCM의 가장 큰 특징이 바로 무결성·인증을 함께 제공한다는 점이므로 틀린 설명이다. 따라서 정답은 ④번이다.

①(암호화에 CTR 모드 사용), ②(인증 태그 생성에 갈루아 필드 기반 해시 연산 사용), ③(CTR 기반이라 블록 병렬 처리 가능)은 모두 옳다.

오답 분석

보기①

옳음. GCM의 암호화 부분은 카운터를 암호화하는 CTR 모드를 사용한다.

보기②

옳음. 인증 태그(GMAC) 생성에 갈루아 필드 GF(2¹²⁸)의 곱셈 기반 해시 연산을 사용한다.

보기③

옳음. CTR 기반이므로 암호화·복호화 모두 블록 병렬 처리가 가능해 고속이다.

보기④

정답(부적절). GCM은 기밀성뿐 아니라 무결성·인증까지 제공하는 AEAD 모드다.

관련 이론 정리

전통적으로 기밀성(암호화)과 무결성(MAC)은 별도로 적용했는데, 결합 순서를 잘못 설계하면 취약점이 생겨(Encrypt-and-MAC, MAC-then-Encrypt의 문제) 인증 암호(AEAD) 모드가 표준이 되었다. GCM은 CTR로 암호화하고 GHASH로 인증 태그를 만들며, 암호화하지 않고 무결성만 보장하는 추가 인증 데이터(AAD)(예: 패킷 헤더)를 함께 처리할 수 있어 TLS 1.3·IPSec·WPA3에서 널리 쓰인다. CCM은 CTR + CBC-MAC 조합의 AEAD이고, ChaCha20-Poly1305는 하드웨어 AES 가속이 없는 환경에서 쓰인다. GCM 사용 시 동일 키에 대해 논스(IV)를 절대 재사용하면 안 되며, 재사용 시 인증 키가 노출되어 위조가 가능해진다. 감리에서는 AEAD 모드 채택과 논스 관리, 태그 검증 여부를 점검한다.

운용 모드와 제공 서비스:

GCM O O(AEAD)
CCM O O(AEAD)

실무 적용 사례

암호 구현 감리 시 CBC 등 기밀성만 제공하는 모드에 별도 MAC 없이 사용하는 구간이 있는지 확인하고, GCM 등 AEAD 전환과 논스 재사용 방지를 권고한다.

감리사 시험 출제 포인트

GCM = CTR(기밀성) + GMAC(무결성)의 AEAD 모드임이 핵심이다. "무결성 미보장"이라는 서술이 전형적 함정이다.

관련 이론 정리

VLAN은 물리적 배선과 무관하게 포트·MAC·프로토콜 기준으로 논리적 그룹을 만들어 브로드캐스트 도메인을 분할한다. 스위치 간에는 IEEE 802.1Q 태그로 VLAN ID(12비트, 최대 4094개)를 프레임에 삽입해 여러 VLAN 트래픽을 하나의 트렁크 포트로 전달한다. VLAN 간 통신은 3계층 장비(라우터·L3 스위치)를 거쳐야 하므로 접근통제 지점(ACL)을 자연스럽게 확보할 수 있고, 부서·업무·보안등급별 분리로 내부 확산을 억제한다. 보안 위협으로는 이중 태깅이나 트렁크 협상을 악용해 다른 VLAN에 접근하는 VLAN 호핑(hopping)이 있어, 미사용 포트 비활성화, DTP 비활성화, 네이티브 VLAN 변경이 권고된다. 감리에서는 VLAN 설계가 업무·보안 등급과 일치하는지, VLAN 간 ACL이 최소권한으로 설정됐는지 점검한다.

정답 해설

Snort 규칙은 액션 프로토콜 출발지IP 출발지포트 → 목적지IP 목적지포트 (옵션) 형식이다. 요구사항을 하나씩 대응시키면 다음과 같다.

관련 이론 정리

Snort는 대표적 오픈소스 IDS/IPS로, 규칙 헤더(액션·프로토콜·주소·포트·방향)와 옵션(msg·content·sid·flags 등)으로 탐지 규칙을 정의한다. 액션은 alert(경고+로그), log(로그만), pass(무시), drop(차단+로그), reject(차단+거부 응답), sdrop(로그 없이 차단)으로 나뉘며, drop 계열은 인라인(IPS) 모드에서만 동작한다. 주소·포트에는 any, CIDR, 부정 연산자 !, 목록 [ ]을 쓸 수 있고 방향 연산자는 단방향 ->와 양방향 <>가 있다. 탐지 방식은 알려진 패턴을 찾는 오용 탐지(signature)와 정상 프로파일과의 편차를 보는 이상 탐지(anomaly)로 구분되며, Snort는 주로 오용 탐지 기반이라 미지의 공격(제로데이)에는 취약하다. 감리에서는 룰셋 최신화, 오탐 관리, IPS 인라인 구성의 가용성 영향을 점검한다.

주소 CIDR, any, !(부정)
방향 -> 단방향, <> 양방향
포트 FTP 21, HTTP 80, SSH 22 등
실무 적용 사례

IDS/IPS 감리 시 탐지 전용(alert)과 차단(drop) 규칙의 구분, 룰셋 업데이트 주기, 오탐으로 인한 정상 서비스 차단 여부를 점검한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

ARP 스푸핑(ARP 포이즈닝)은 공격자가 위조된 ARP 응답을 보내 피해자의 ARP 캐시에 잘못된 IP-MAC 매핑을 심는 공격이다. 예컨대 게이트웨이의 IP에 공격자의 MAC을 대응시키면 피해자의 모든 외부 트래픽이 공격자를 거치게 되어 중간자 공격(MITM)으로 도청·변조가 가능해진다. 문제의 "IP 주소와 MAC 주소의 매핑 정보를 위조"라는 서술과 정확히 일치하므로 정답은 ①번이다.

②IP 스푸핑은 출발지 IP 주소를 위조하는 공격, ③ICMP 리다이렉트는 라우팅 경로를 변경하도록 유도하는 공격, ④DNS 스푸핑은 도메인-IP 매핑을 위조하는 공격으로 각각 위조 대상이 다르다.

오답 분석

보기①

정답. IP-MAC 매핑을 위조하는 것이 ARP 스푸핑이다.

보기②

IP 스푸핑은 패킷의 출발지 IP를 위조해 신분을 속이는 공격이다.

보기③

ICMP 리다이렉트는 잘못된 경로 정보를 보내 트래픽 경로를 바꾸는 공격이다.

보기④

DNS 스푸핑은 도메인 이름에 대응하는 IP를 위조하는 공격이다.

관련 이론 정리

ARP는 같은 LAN에서 IP를 MAC으로 변환하는 프로토콜인데, 인증 절차가 없어 위조 응답을 그대로 캐시에 반영한다. 이를 악용한 ARP 스푸핑은 같은 브로드캐스트 도메인 내에서만 가능하며, 성공하면 스니핑(도청), 세션 하이재킹, 트래픽 변조로 이어진다. 유사 기법으로 스위치의 MAC 테이블을 가득 채워 허브처럼 동작하게 만드는 스위치 재밍(MAC 플러딩), 라우터로 위장하는 ARP 리다이렉트가 있다. 대응책은 중요 장비에 대한 정적(static) ARP 등록, 스위치의 DAI(Dynamic ARP Inspection)와 DHCP 스누핑 연계, ARP 캐시 이상 탐지, 그리고 근본적으로 종단 간 암호화(HTTPS·VPN) 적용이다. 감리에서는 내부망 스니핑 위험에 대한 통제와 중요 구간 암호화 적용을 점검한다.

스푸핑 기법별 위조 대상:

기법 위조 대상
ARP 스푸핑 IP ↔ MAC 매핑
IP 스푸핑 출발지 IP 주소
DNS 스푸핑 도메인 ↔ IP 매핑
ICMP 리다이렉트 라우팅 경로
실무 적용 사례

내부망 보안 감리 시 DAI·DHCP 스누핑 설정, 중요 서버의 정적 ARP 등록, 내부 구간 평문 통신(HTTP·Telnet) 사용 여부를 점검한다.

정답 해설

SSO는 인증을 중앙의 SSO 서버로 집중시켜 한 번의 로그인으로 여러 시스템을 이용하게 하는 방식이다. 따라서 SSO 서버에 장애가 나면 연결된 모든 시스템의 인증이 불가능해지는 단일 장애점(SPOF) 문제가 존재하며, 이를 완화하려면 SSO 서버를 이중화·클러스터링해야 한다.

④는 "각 시스템마다 인증 정보를 저장하므로 SPOF 문제가 존재하지 않는다"고 서술했는데, 이는 SSO가 아니라 개별 인증 방식의 특성이며 SSO의 구조와 정반대이므로 틀렸다. 따라서 정답은 ④번이다.

①(한 번 인증으로 관리 대상 시스템 접근), ②(Kerberos 기반 윈도우 AD가 대표 사례), ③(중요 정보 접근 시 단계적 재인증 요구)은 모두 옳다.

관련 이론 정리

SSO는 사용자 편의성과 계정 관리 효율을 높이지만 보안 측면에서 양면성이 있다. 장점은 비밀번호 재사용·메모 부착 감소, 계정 생명주기의 중앙 관리, 접근 로그 일원화다. 단점은 SPOF와 함께, 하나의 인증정보가 유출되면 연결된 모든 시스템이 한꺼번에 위험해지는 점(키 마스터 문제)이다. 그래서 다중 인증(MFA) 결합, 세션 타임아웃, 중요 자원 접근 시 재인증이 권장된다. 구현 기술로 Kerberos(티켓 기반, TGT/TGS), SAML(웹 SSO의 XML 기반 표준), OAuth 2.0(권한 위임)과 그 위에 인증을 얹은 OpenID Connect가 있으며, OAuth는 인증이 아니라 인가(권한 위임) 프로토콜이라는 점을 구분해야 한다. EAM/IAM은 SSO에 권한 관리와 계정 수명주기 관리를 더한 체계다. 감리에서는 SSO 이중화, MFA 적용, 세션 관리 정책을 점검한다.

정답 해설

③은 두 가지 점에서 틀렸다.

첫째, 관리 키 개수: n명이 통신할 때 대칭키는 n(n−1)/2개의 키가 필요한 반면, 공개키는 각자 키 쌍 하나씩 2n개면 되므로 사용자가 많아질수록 공개키 방식이 관리할 키가 훨씬 적다. 둘째, 속도: 공개키 연산은 큰 수의 멱승·타원곡선 연산이라 대칭키보다 수백~수천 배 느려 대용량 메시지 암호화에는 부적합하다. 실제로는 대칭키가 메시지 암호화를 담당하고 공개키는 세션키 전달·전자서명에 쓰인다.

따라서 "키 개수가 많고 속도가 빨라 메시지 암호화에 주로 사용한다"는 ③은 모두 반대이며 정답은 ③번이다. ①②④는 모두 옳다.

관련 이론 정리

대칭키와 공개키는 상호 보완적이다. 대칭키(AES·SEED·ARIA)는 속도가 빨라 대용량 데이터에 적합하지만 키 배송 문제와 사용자 수 증가 시 키 개수 폭증(n(n−1)/2) 문제가 있다. 공개키(RSA·ECC)는 키 배송 문제를 해결하고 전자서명·부인방지를 제공하지만 느리다. 그래서 실무는 하이브리드 방식을 쓴다: 임의의 세션키를 생성해 대칭키로 데이터를 암호화하고, 그 세션키를 수신자의 공개키로 암호화해 함께 전달한다(TLS·PGP·S/MIME이 모두 이 구조). 기밀성은 "수신자 공개키로 암호화", 전자서명은 "송신자 개인키로 서명"이라는 키 사용 방향의 구분이 핵심이며 시험에서 자주 뒤바뀐다. 감리에서는 키 길이·알고리즘 적정성과 키 관리(생성·보관·폐기) 절차를 점검한다.

감리사 시험 출제 포인트

공개키 = 키 개수 적음 + 속도 느림, 대칭키 = 키 개수 많음 + 속도 빠름의 대비가 핵심이다. 두 특성을 뒤바꾼 오답이 전형적이다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

HMAC은 송·수신자가 공유한 비밀키와 해시 함수로 인증코드를 만들어 무결성과 (상호 간의) 메시지 인증을 제공한다. 그러나 비밀키를 양측이 동일하게 보유하므로 수신자도 동일한 MAC을 만들 수 있어, 제3자 입장에서는 그 메시지를 송신자가 만들었는지 수신자가 만들었는지 구별할 수 없다. 따라서 HMAC은 부인방지(non-repudiation)를 제공하지 못한다.

③은 "송신자가 부인할 때 수신자가 송신 사실을 증명할 수 있다"고 서술했는데 이는 부인방지에 해당하며, HMAC이 아니라 송신자의 개인키로 서명하는 전자서명만이 제공한다. 따라서 정답은 ③번이다. ①②④는 모두 옳다.

오답 분석

보기①

옳음. HMAC으로 메시지 변조 여부와 키를 아는 송신자가 만든 메시지임을 확인할 수 있다.

보기②

옳음. 양자 간 인증은 제공하지만 제3자에 대한 증명(부인방지)은 불가하다.

보기③

정답(부적절). 부인방지는 전자서명의 기능이며 HMAC은 제공하지 못한다.

보기④

옳음. HMAC은 사전에 공유된 대칭 비밀키를 필요로 한다.

관련 이론 정리

메시지 인증 수단은 제공 서비스로 구분해야 한다. 해시(단독)는 무결성만 제공하고 키가 없어 위조가 가능하므로 인증은 못 한다. MAC/HMAC은 대칭키를 써서 무결성 + 인증을 제공하지만, 키를 공유하는 구조상 부인방지는 불가하다. 전자서명은 송신자의 개인키로 서명하고 누구나 공개키로 검증할 수 있어 무결성·인증·부인방지를 모두 제공한다. HMAC은 해시 함수에 키를 단순 결합할 때 생기는 길이 확장 공격을 막기 위해 내부·외부 두 번의 해시 연산(ipad·opad) 구조를 사용하며, HMAC-SHA256 등으로 표기한다. 실무에서는 API 요청 서명, TLS 무결성 검증, JWT의 HS256 서명에 널리 쓰인다. 감리에서는 부인방지가 요구되는 거래에 MAC이 아닌 전자서명이 적용됐는지 점검한다.

수단 무결성 인증 부인방지
해시 O X X
HMAC O O X
전자서명 O O O
실무 적용 사례

전자거래 감리 시 법적 분쟁 가능성이 있는 거래에 전자서명(공인·공동인증서 등)이 적용됐는지, 단순 HMAC으로 대체해 부인방지가 결여되지 않았는지 점검한다.

감리사 시험 출제 포인트

HMAC은 부인방지를 제공하지 않는다는 점이 최다 출제 포인트다. 부인방지=전자서명(개인키)로 짝지어 암기한다.

정답 해설

하이브리드 암호 시스템의 구조는 "메시지는 대칭키(세션키)로, 세션키는 수신자의 공개키로" 암호화하는 것이다. 공개키 연산이 느리기 때문에 대용량 메시지에는 쓰지 않고, 짧은 세션키만 공개키로 보호해 속도와 키 분배 문제를 동시에 해결한다.

②는 "메시지를 암호화할 때 공개키 암호 시스템을 이용한다"고 서술했는데, 메시지 암호화는 대칭키가 담당하므로 틀렸다. 따라서 정답은 ②번이다.

①(세션키는 예측 불가하도록 안전한 난수 생성기로 생성), ③(세션키를 수신자 공개키로 암호화해 암호문과 함께 전송), ④(PGP가 대표적 하이브리드 구현)는 모두 옳다.

오답 분석

보기②

정답(부적절). 메시지 암호화는 대칭키가 담당하며 공개키는 세션키 보호에 쓰인다.

보기③

옳음. 세션키를 수신자 공개키로 암호화해 암호문과 결합·전송한다.

보기④

옳음. PGP는 하이브리드 암호 구조를 사용하는 대표적 메일 암호화 도구다.

관련 이론 정리

하이브리드 암호는 대칭키의 속도와 공개키의 키 분배 해결·부인방지를 결합한 구조다. 송신 측은 ① 임의의 세션키 생성 → ② 세션키로 메시지 암호화 → ③ 수신자 공개키로 세션키 암호화 → ④ 암호문 + 암호화된 세션키 전송의 순서로 처리하고, 수신 측은 자신의 개인키로 세션키를 복원한 뒤 메시지를 복호화한다. PGP는 여기에 송신자 개인키 서명(부인방지)과 압축을 더하고, S/MIME은 인증서 기반으로 유사 구조를 제공하며, TLS도 핸드셰이크로 세션키를 만들고 이후 대칭키로 통신한다. 세션키는 매 세션마다 새로 생성되어야 하며, 재사용하거나 예측 가능한 난수를 쓰면 전체 보안이 무너진다. 감리에서는 난수 생성기의 안전성과 키 폐기 절차를 점검한다.

하이브리드 암호 처리 순서:

단계 처리
1 세션키(대칭키) 생성
2 메시지 → 세션키로 암호화
3 세션키 → 수신자 공개키로 암호화
4 암호문 + 암호화된 세션키 전송
실무 적용 사례

메일·파일 암호화 감리 시 대용량 데이터를 공개키로 직접 암호화하는 비효율 구현이 없는지, 세션키가 매번 새로 생성되는지 점검한다.

정답 해설

생체 인증은 등록(enrollment) 단계에서 사용자의 생체 특징을 추출해 생체 템플릿 형태로 저장해 두고, 인증 시 새로 측정한 값과 저장된 템플릿을 대조해 본인 여부를 판단한다. 즉 사전 등록 없이는 비교 기준이 없어 인증 자체가 불가능하다.

③은 "생체 정보를 데이터베이스 등에 등록 없이 인증이 가능하다"고 서술했으므로 틀렸다. 따라서 정답은 ③번이다.

①(지식 기반은 저비용이나 추측·유출 위험), ②(소유 기반은 복제·분실·파손 위험), ④(서로 다른 두 요소 결합 = 이중 인증)는 모두 옳다.

보기①

보기②

옳음. 토큰·스마트카드 등 소유 기반은 복제·분실·파손 시 인증이 불가하다.

보기③

정답(부적절). 생체 인증은 사전 등록(템플릿 저장)이 반드시 필요하다.

보기④

옳음. 서로 다른 두 가지 인증 요소를 결합하면 이중(Two-factor) 인증이다.

관련 이론 정리

인증 요소는 지식 기반(what you know: 비밀번호·PIN), 소유 기반(what you have: OTP·스마트카드·인증서), 존재/생체 기반(what you are: 지문·홍채·얼굴), 행위 기반(what you do: 서명·타이핑 패턴·음성)으로 나뉜다. 이 중 서로 다른 범주의 요소를 둘 이상 결합해야 진정한 다중 인증이며, 비밀번호 두 개를 요구하는 것은 이중 인증이 아니다. 생체 인증은 분실·망각이 없고 편리하지만, 유출 시 변경이 불가능하고(비밀번호와 결정적 차이) 측정 오차로 FAR(타인수락률)과 FRR(본인거부율)이 발생하며 둘이 같아지는 지점이 EER(동등오류율)로 성능 지표가 된다. 생체 템플릿은 개인정보 중 민감정보로 분류되어 암호화 저장·분리 보관이 요구된다. 감리에서는 다중 인증 적용 범위와 생체정보 보호조치를 점검한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

침입탐지 결과는 실제 상태와 판정의 조합으로 네 가지로 나뉜다. 이 중 False-Positive(오탐)는 정상 트래픽을 공격으로 잘못 판정하는 것으로, IPS가 이를 차단하면 정상 사용자의 서비스 이용이 막혀 가용성이 침해된다. 문제 설명과 정확히 일치하므로 정답은 ③번이다.

①True-Positive는 공격을 공격으로 정확히 탐지한 정상 동작, ②True-Negative는 정상을 정상으로 판정한 정상 동작, ④False-Negative(미탐)는 실제 공격을 정상으로 잘못 판정해 침해를 놓치는 경우로 기밀성·무결성 사고로 이어진다.

오답 분석

보기①

True-Positive는 실제 공격을 정확히 탐지한 경우로 오류가 아니다.

보기②

True-Negative는 정상 트래픽을 정상으로 정확히 판정한 경우다.

보기③

정답. 정상을 공격으로 오인하는 오탐으로 가용성이 침해된다.

보기④

False-Negative는 공격을 놓치는 미탐으로, 가용성이 아니라 침해사고로 이어진다.

관련 이론 정리

IDS/IPS 성능은 오탐과 미탐의 트레이드오프로 결정된다. 탐지 임계값을 낮추면(민감하게) 미탐은 줄지만 오탐이 늘어 정상 서비스가 차단되고, 임계값을 높이면 오탐은 줄지만 미탐이 늘어 공격을 놓친다. 그래서 운영 초기에는 탐지 전용(alert) 모드로 튜닝한 뒤 차단(drop) 모드로 전환하는 것이 일반적이다. 탐지 방식별로도 특성이 다른데, 오용 탐지(시그니처 기반)는 알려진 공격에 정확도가 높아 오탐이 적지만 미지의 공격(제로데이)에 미탐이 발생하고, 이상 탐지(행위 기반)는 새로운 공격을 잡을 수 있지만 정상 행위의 변동으로 오탐이 많다. 성능 지표로는 정탐률(재현율)·정밀도·ROC 곡선이 쓰인다. 감리에서는 룰 튜닝 절차, 오탐 처리 이력, 차단 정책의 서비스 영향 검토를 점검한다.

탐지 결과 4분면:

판정\실제 공격 정상
공격으로 판정 True-Positive(정탐) False-Positive(오탐)
정상으로 판정 False-Negative(미탐) True-Negative
실무 적용 사례

IPS 운영 감리 시 차단 정책 적용 전 오탐 튜닝 기간을 두었는지, 오탐으로 인한 서비스 장애 이력과 예외 처리 절차가 관리되는지 점검한다.

감리사 시험 출제 포인트

오탐(False-Positive) → 가용성 침해, 미탐(False-Negative) → 침해사고의 연결이 핵심이다. 용어를 바꾼 오답이 전형적이다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

①②③은 모두 블록체인 환경에 특화된 공격이다. 이중 지불 공격은 동일 코인을 두 번 사용하려는 시도로 51% 공격 등과 연계되고, Eclipse 공격은 특정 노드의 연결을 공격자 노드로 채워 네트워크에서 고립시켜 허위 정보를 주입하는 공격이며, 재진입(Reentrancy) 공격은 스마트컨트랙트가 외부 호출 후 상태를 갱신하기 전에 재귀적으로 다시 호출되어 자금이 반복 인출되는 취약점이다(The DAO 사건).

반면 ④스머프 공격은 출발지를 피해자로 위조한 ICMP 요청을 브로드캐스트로 보내 증폭된 응답으로 대역을 고갈시키는 전통적 DDoS 기법으로, 블록체인 특화 공격이 아니다. 따라서 정답은 ④번이다.

오답 분석

보기①

이중 지불 공격은 블록체인의 합의 특성을 노린 대표적 공격이다.

보기②

Eclipse 공격은 노드를 고립시켜 잘못된 체인 정보를 주입하는 블록체인 공격이다.

보기③

재진입 공격은 스마트컨트랙트의 대표적 취약점을 악용한 블록체인 공격이다.

보기④

정답(부적절). 스머프 공격은 ICMP 증폭 기반의 전통적 DDoS로 블록체인 특화 공격이 아니다.

관련 이론 정리

블록체인 보안 위협은 계층별로 나뉜다. 합의 계층에서는 해시파워 과반을 확보해 거래를 되돌리는 51% 공격과 그로 인한 이중 지불, 채굴 보상을 노린 이기적 채굴이 있다. 네트워크 계층에서는 피해 노드의 연결을 장악하는 Eclipse 공격, 트래픽을 분할하는 라우팅 공격이 있다. 애플리케이션(스마트컨트랙트) 계층에서는 재진입, 정수 오버플로, 접근제어 누락, 난수 예측 취약점이 대표적이며, 배포 후 코드 수정이 어렵기 때문에 사전 감사(audit)와 정형 검증이 중요하다. 키 관리 계층에서는 개인키 분실·탈취가 곧 자산 상실로 이어진다. 한편 스머프·SYN 플러딩 등 전통적 DDoS는 블록체인 노드에도 적용될 수 있으나 블록체인 '특화' 공격으로 분류되지는 않는다. 감리에서는 스마트컨트랙트 감사 이력과 키 관리 체계를 점검한다.

블록체인 공격 유형:

계층 공격
합의 51% 공격, 이중 지불
네트워크 Eclipse, 라우팅 공격
스마트컨트랙트 재진입, 정수 오버플로
(전통 DDoS) 스머프·SYN 플러딩
실무 적용 사례

블록체인 사업 감리 시 스마트컨트랙트 외부 감사 수행 여부, 재진입 방지 패턴(Checks-Effects-Interactions) 적용, 개인키 보관(HSM·다중서명) 체계를 점검한다.

감리사 시험 출제 포인트

이중 지불·Eclipse·재진입은 블록체인 특화, 스머프는 전통 DDoS라는 구분이 핵심이다.
