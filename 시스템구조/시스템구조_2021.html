영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

썬더볼트(Thunderbolt)는 인텔과 애플이 개발한 초고속 주변장치 접속 인터페이스로, PCI Express(데이터)와 DisplayPort(영상)를 하나의 케이블로 통합하고 전력 공급까지 함께 제공한다. 외장 스토리지·RAID 배열·비디오 캡처 장비·이더넷 어댑터 등을 데이지체인으로 연결할 수 있어 문제 설명과 정확히 일치한다. 따라서 정답은 ④번이다.

①IvyBridge와 ③SandyBridge는 인텔 CPU 마이크로아키텍처 코드명이고, ②QPI는 CPU와 칩셋·다른 CPU를 잇는 시스템 내부 인터커넥트로, 모두 외부 주변장치 접속 기술이 아니다.

오답 분석

보기①

IvyBridge는 인텔 CPU 마이크로아키텍처(22nm) 코드명으로 접속 기술이 아니다.

보기②

QPI는 CPU 간·CPU-칩셋 간 고속 내부 연결 버스로 외부 주변장치용이 아니다.

보기③

SandyBridge도 인텔 CPU 마이크로아키텍처 코드명이다.

보기④

정답. 썬더볼트는 데이터·비디오·오디오·전력을 통합한 외부 주변장치 접속 기술이다.

관련 이론 정리

시스템의 연결 기술은 위치에 따라 구분된다. 내부 인터커넥트는 CPU·메모리·칩셋을 잇는 것으로 QPI(이후 UPI), HyperTransport, PCI Express(확장 카드) 등이 있다. 외부 주변장치 인터페이스는 USB, Thunderbolt, eSATA, HDMI/DisplayPort 등이 있다. 썬더볼트는 버전이 올라가며 대역폭이 크게 증가했고(TB3/TB4는 40Gbps), USB-C 커넥터를 채택해 USB4와 호환성을 갖췄다. PCIe를 외부로 확장하는 특성 덕분에 외장 GPU·고속 스토리지에도 쓰이지만, 그만큼 DMA 공격 위험이 있어 보안 설정(장치 승인)이 필요하다. 감리에서는 고속 인터페이스 도입 시 대역폭·호환성과 외부 포트 보안 통제를 점검한다.

연결 기술 구분:

구분 기술
CPU 내부/시스템 QPI·UPI, HyperTransport, PCIe
외부 주변장치 Thunderbolt, USB, eSATA
CPU 아키텍처 SandyBridge, IvyBridge
실무 적용 사례

고성능 워크스테이션·영상 시스템 도입 감리 시 인터페이스 대역폭이 요구 성능을 충족하는지, 외부 포트를 통한 DMA·데이터 유출 통제가 있는지 점검한다.

감리사 시험 출제 포인트

CPU 아키텍처 코드명(SandyBridge·IvyBridge)과 접속 기술(Thunderbolt·QPI)을 섞어 놓는 구성이다. "데이터+영상+전력 통합"이 썬더볼트의 단서다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

컨테이너 이미지는 게스트 운영체제를 포함하지 않고 애플리케이션과 필요한 라이브러리·실행 파일만 담기 때문에, 전체 OS를 포함하는 가상머신 이미지보다 훨씬 작다(보통 수십~수백 MB vs 수 GB). 또한 레이어 구조로 공통 레이어를 공유해 저장 공간을 더 절약한다.

③은 "이미지 공간이 가상머신 방식보다 더 많이 필요하다"고 서술했으나 실제로는 더 적게 필요하므로 틀린 설명이다. 따라서 가장 적절하지 않은 것은 ③번이다. ①(프로세스 단위 격리), ②(커널 공유·앱과 라이브러리만), ④(게스트 OS·하이퍼바이저 없어 성능 손실 적음)는 모두 옳다.

오답 분석

보기①

옳음. 컨테이너는 네임스페이스·cgroup으로 프로세스 단위 격리 환경을 만든다.

보기②

옳음. 호스트 커널을 공유하고 애플리케이션 실행에 필요한 요소만 포함한다.

보기③

정답(부적절). 컨테이너 이미지는 VM 이미지보다 작다.

보기④

옳음. 게스트 OS·하이퍼바이저 계층이 없어 오버헤드가 작다.

관련 이론 정리

도커 컨테이너 이미지는 읽기 전용 레이어의 스택으로 구성되며, 실행 시 최상단에 쓰기 가능 레이어를 얹는다(Union File System). 여러 이미지가 동일한 베이스 레이어(예: alpine, ubuntu 기본 이미지)를 공유하므로 디스크 사용과 배포 트래픽이 크게 절약된다. VM은 각 인스턴스가 완전한 게스트 OS를 포함해 이미지가 크고 부팅에 수십 초가 걸리는 반면, 컨테이너는 수 초 내 기동한다. 다만 커널을 공유하므로 호스트와 다른 커널이 필요한 OS는 실행할 수 없고(리눅스 호스트에서 윈도 컨테이너 불가), 격리 강도가 VM보다 약하다. 이미지 보안을 위해 취약점 스캔·최소 베이스 이미지·비루트 실행이 권장된다. 감리에서는 이미지 관리(레지스트리·태깅)와 보안 스캔 체계를 점검한다.

실무 적용 사례

컨테이너 도입 감리 시 이미지 크기 최적화(멀티스테이지 빌드), 레지스트리 관리, 취약점 스캔·비루트 실행 등 보안 통제를 점검한다.

감리사 시험 출제 포인트

컨테이너 이미지가 VM보다 작다는 점이 핵심 함정이다. 커널 공유·프로세스 격리·빠른 기동도 함께 정리한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

혼동행렬 기반 지표의 올바른 정의는 다음과 같다.

  • 정확도(Accuracy) = (TP+TN)/(TP+TN+FP+FN) → ①은 분자를 (TP+FP)로 써서 틀림
  • 정밀도(Precision) = TP/(TP+FP) → ②는 TN/(TN+FN)으로 써서 틀림(이는 음성 예측 정확도에 가까움)
  • F1-점수 = 2·Precision·Recall/(Precision+Recall) = 2TP/(2TP+FP+FN) → ③은 분자·분모에 FP를 잘못 배치해 틀림
  • 재현율(Recall) = TP/(TP+FN) → ④는 정확히 일치

따라서 올바르게 정의된 것은 ④번이다. 재현율은 실제 양성 중 모델이 양성으로 맞힌 비율로, 놓치면 안 되는 문제(질병 진단·이상 탐지)에서 특히 중요하다.

오답 분석

보기①

정확도의 분자는 올바르게 맞힌 (TP+TN)이어야 한다. (TP+FP)는 양성 예측 건수다.

보기②

정밀도는 TP/(TP+FP)다. TN/(TN+FN)은 음성 예측의 정확도(NPV)에 해당한다.

보기③

F1은 2TP/(2TP+FP+FN)이다. 분자에 FP가 오는 것은 잘못이다.

보기④

정답. 재현율 = TP/(TP+FN)이 정확한 정의다.

관련 이론 정리

혼동행렬은 TP(실제 양성-예측 양성), FN(실제 양성-예측 음성), FP(실제 음성-예측 양성), TN(실제 음성-예측 음성)으로 구성된다. 정밀도는 "양성이라 한 것 중 맞은 비율"로 오탐(FP)을 줄이는 데, 재현율은 "실제 양성 중 찾아낸 비율"로 미탐(FN)을 줄이는 데 초점을 둔다. 두 지표는 임계값 조정에 따라 상충하므로 조화평균인 F1-score로 균형을 본다. 특이도는 TN/(TN+FP)로 실제 음성을 음성으로 판정한 비율이다. 클래스 불균형이 심하면 정확도는 다수 클래스에 좌우되어 왜곡되므로 정밀도·재현율·F1·AUC를 함께 봐야 한다. 감리에서는 업무 특성(오탐과 미탐 중 무엇이 더 치명적인지)에 맞는 지표 선택과 임계값 설정의 타당성을 점검한다.

지표 공식 정리:

지표 공식
Accuracy (TP+TN)/전체
Precision TP/(TP+FP)
Recall TP/(TP+FN)
Specificity TN/(TN+FP)
F1 2TP/(2TP+FP+FN)
실무 적용 사례

이상탐지·의료 진단 AI 감리 시 미탐(FN)이 치명적인 업무에서 재현율 목표를 우선했는지, 불균형 데이터에서 정확도만으로 평가하지 않았는지 검토한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

이동 IP(Mobile IP)에서 홈 주소(home address)는 이동 호스트의 영구적이고 변하지 않는 식별자다. 호스트가 다른 네트워크로 이동해도 홈 주소는 그대로 유지되며, 대신 이동한 지역에서 CoA(Care of Address)라는 임시 주소를 추가로 받는다. 홈 에이전트는 홈 주소로 온 패킷을 가로채 터널링(캡슐화)으로 CoA에 전달한다.

②는 "홈 주소가 위치가 바뀌면 새로 할당되어야 한다"고 서술했으나, 새로 할당되는 것은 CoA이고 홈 주소는 불변이므로 틀린 설명이다. 따라서 정답은 ②번이다. ①③④는 모두 옳다.

보기①

옳음. 홈 에이전트가 받은 패킷을 이동 호스트(CoA)로 보내려면 터널이 필요하다.

보기②

정답(부적절). 위치 변경 시 새로 할당되는 것은 CoA이며 홈 주소는 고정이다.

보기③

옳음. CoA는 이동한 지역에서 일시적으로 할당되는 주소다.

보기④

옳음. 터널링은 원래 IP 패킷을 다른 IP 헤더로 캡슐화해 중개한다.

관련 이론 정리

이동 IP는 호스트가 네트워크를 옮겨도 동일한 IP(홈 주소)로 연결을 유지하게 하는 기술이다. 구성요소는 이동 호스트, 홈 네트워크에서 대리 수신하는 홈 에이전트(HA), 방문 네트워크의 외부 에이전트(FA)다. 이동 호스트는 새 위치에서 CoA를 얻어 홈 에이전트에 등록하고, 홈 주소로 오는 패킷은 홈 에이전트가 가로채 IP-in-IP 캡슐화로 CoA까지 터널링한 뒤 원본 패킷을 꺼내 전달한다. 이때 송신자→홈→이동호스트로 우회하는 삼각 라우팅(triangle routing) 비효율이 발생해, 경로 최적화 기법이 제안됐다. 터널링은 이동 IP 외에도 VPN·IPv6 전환(6to4)에서 널리 쓰인다. 감리에서는 이동성 지원 설계와 터널 오버헤드·보안(암호화)을 점검한다.

이동 IP 주소 체계:

주소 성격
홈 주소 영구·불변 식별자
CoA 방문 네트워크의 임시 주소
실무 적용 사례

모바일·무선 네트워크 감리 시 이동성 지원 방식, 터널링 오버헤드로 인한 성능 저하, 터널 구간 암호화 적용을 점검한다.

② 직접 사상 캐시를 사용할 때, 일반적으로 LRU(Least Recently Used) 교체 정책의 캐시 적중률이 임의(random) 교체 정책보다 우수하다.

정답 해설

완전 연관 캐시는 메모리 블록을 캐시의 어느 라인에나 배치할 수 있어 적중률이 높지만, 특정 블록을 찾으려면 모든 라인의 태그를 확인해야 한다. 순차적으로 검색하면 라인 수에 비례해 시간이 걸리는 반면, 직접 사상 캐시는 인덱스로 한 곳만 확인하면 되므로 검색이 가장 빠르다. 따라서 ④의 "완전 연관 캐시의 순차 검색 시간이 직접 사상보다 길다"는 옳은 설명이며 정답은 ④번이다.

관련 이론 정리

캐시 사상 방식은 메모리 블록을 캐시 어디에 둘지 정하는 규칙이다. 직접 사상은 (블록번호 mod 캐시 라인 수)로 위치가 하나로 결정되어 하드웨어가 단순하고 검색이 빠르지만, 같은 라인에 매핑되는 블록들이 번갈아 쓰이면 충돌 미스(conflict miss)가 잦다. 완전 연관은 어디든 배치 가능해 충돌 미스가 없지만 모든 태그를 비교해야 해(보통 CAM으로 병렬 비교) 하드웨어가 비싸고 느리며, 교체 정책(LRU 등)이 필요하다. 집합 연관은 둘의 절충으로 캐시를 여러 집합으로 나누고 집합 내에서는 완전 연관처럼 동작해(예: 4-way) 실무 표준이 됐다. 캐시 미스는 강제(cold)·용량(capacity)·충돌(conflict) 미스로 분류된다. 감리에서는 성능 분석 시 캐시 적중률과 메모리 접근 패턴을 검토한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

MTTR(평균 수리 시간)은 총 수리(장애) 시간을 장애 발생 건수로 나눈 값이다.

  • 월요일 장애: 13:00~14:00 = 1시간
  • 목요일 장애: 16:00~19:00 = 3시간
  • 총 장애 시간 = 1 + 3 = 4시간, 장애 건수 = 2건

MTTR = 4시간 ÷ 2건 = 2시간

따라서 정답은 ②번이다. ④(4시간)는 총 장애 시간을 그대로 답한 대표적 오답으로, 건수로 나누는 평균임을 놓친 경우다.

오답 분석

보기①

보기②

정답. 총 4시간 ÷ 2건 = 2시간.

보기③

3시간은 목요일 장애 시간만 본 값이다.

보기④

4시간은 총 장애 시간으로, 건수로 나누지 않은 값이다.

관련 이론 정리

신뢰성·가용성 지표는 서로 맞물려 정의된다. MTBF(Mean Time Between Failures)는 고장과 고장 사이의 평균 정상 가동 시간으로 총 가동시간 ÷ 고장 건수이고, MTTF는 수리 불가 제품의 평균 수명, MTTR(Mean Time To Repair)는 총 수리시간 ÷ 고장 건수다. 가용성(Availability) = MTBF / (MTBF + MTTR)로 계산되므로, 가용성을 높이려면 MTBF를 늘리거나(품질·이중화) MTTR을 줄여야(신속 복구·자동 장애조치) 한다. 본 문제에서 총 운영시간은 5일×24시간=120시간, 장애 4시간이므로 가용성은 116/120 ≈ 96.7%로 계산할 수도 있다. 감리에서는 SLA의 MTTR·가용성 목표와 실제 장애 이력의 부합 여부, 복구 절차의 실효성을 점검한다.

신뢰성 지표:

지표 계산
MTBF 총 정상가동시간 ÷ 고장 건수
MTTR 총 수리시간 ÷ 고장 건수
가용성 MTBF/(MTBF+MTTR)
실무 적용 사례

운영 감리 시 장애 이력에서 MTTR·MTBF를 산출해 SLA 목표 달성 여부를 평가하고, 복구 시간 단축을 위한 자동화·이중화를 권고한다.

감리사 시험 출제 포인트

MTTR은 건수로 나눈 평균임을 명심한다. 총 장애 시간(4시간)을 답으로 고르는 함정이 반드시 포함된다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

각 레벨에서 가용 용량 3TB를 만들기 위해 필요한 1TB 디스크 수를 계산한다.

  • ① RAID 0(스트라이핑, 중복 없음): 가용 = N → 3개
  • ② RAID 0+1(스트라이핑 + 미러링): 절반이 복제본이므로 가용 = N/2 → 6개
  • ③ RAID 4(전용 패리티 1개): 가용 = N−1 → 4개
  • ④ RAID 6(이중 패리티): 가용 = N−2 → 5개

따라서 가장 많은 디스크가 필요한 것은 RAID 0+1(6개)이고 정답은 ②번이다. 미러링은 용량 효율이 50%로 패리티 방식보다 낮기 때문이다.

오답 분석

보기①

RAID 0은 중복이 없어 3개면 충분하다(가장 적음).

보기②

정답. RAID 0+1은 미러링으로 용량 효율 50%라 6개가 필요하다.

보기③

RAID 4는 패리티 디스크 1개 추가로 4개면 된다.

보기④

RAID 6는 패리티 2개로 5개가 필요하다(6개보다 적음).

관련 이론 정리

RAID 레벨의 용량 효율은 중복 방식에 따라 결정된다. 미러링 계열(RAID 1, 0+1, 10)은 데이터를 통째로 복제하므로 효율이 50%로 가장 낮지만 성능(읽기)과 복구 속도가 우수하다. 패리티 계열(RAID 4·5는 N−1, RAID 6는 N−2)은 디스크 수가 많을수록 효율이 높아지지만(예: 10개 RAID 5는 90%), 쓰기 패널티가 크고 재구축 시간이 길다. RAID 0은 중복이 없어 효율 100%지만 내결함성이 전혀 없다. 실무 선택은 용량 효율·성능·가용성의 삼각 절충으로, 쓰기가 많고 복구 속도가 중요하면 RAID 10, 용량 효율이 중요하고 읽기 위주면 RAID 5/6을 택한다. 감리에서는 요구 용량·가용성 대비 디스크 수 산정이 적정한지, 과다·과소 구성이 아닌지 점검한다.

레벨 가용 용량 필요 디스크
RAID 0 N 3
RAID 4 N−1 4
RAID 6 N−2 5
RAID 0+1 N/2 6
실무 적용 사례

스토리지 용량 산정 감리 시 RAID 레벨별 실제 가용 용량(오버헤드)을 반영해 디스크 수와 예산이 적정하게 산정됐는지 검토한다.

감리사 시험 출제 포인트

미러링=효율 50%(디스크 2배 필요), 패리티=N−1 또는 N−2를 공식화해 암기하면 계산형 문항을 빠르게 해결할 수 있다.

정답 해설

회선교환은 통신 전에 전용 회선을 설정해 일정한 대역폭을 독점하므로 종단 간 전송률(및 지연)을 보장할 수 있다. ③이 이를 정확히 서술하므로 정답은 ③번이다.

관련 이론 정리

회선교환과 패킷교환의 근본 차이는 자원 할당 방식이다. 회선교환은 통신 시작 전 경로와 대역폭을 예약(고정 할당)하므로 지연이 일정하고 전송률이 보장돼 실시간 음성에 적합하지만, 통화 중 말이 없는 시간에도 회선을 점유해 활용률이 낮다. 패킷교환은 예약 없이 패킷 단위로 보내며 여러 사용자의 트래픽을 통계적 다중화로 섞어 링크를 채우므로 활용률이 높고 버스트성 데이터에 적합하지만, 혼잡 시 큐잉 지연·지터·손실이 발생해 품질 보장이 어렵다(QoS·대역 예약 기법으로 보완). 인터넷은 패킷교환(데이터그램) 기반이며, 실시간 서비스는 QoS·우선순위 큐로 품질을 관리한다. 감리에서는 트래픽 특성에 맞는 전송 방식과 QoS 설계를 점검한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

합성곱 출력(특징 맵) 크기 공식은 다음과 같다.

출력 크기 = (N − F + 2P) / S + 1
(N=입력 크기, F=필터 크기, P=패딩, S=스트라이드)

문제 조건은 N=5, F=3, P=0(패딩 없음), S=1이므로,

출력 = (5 − 3 + 0)/1 + 1 = 2 + 1 = 3

따라서 특징 맵은 3×3이고 정답은 ②번이다. 패딩을 적용하지 않으면 합성곱을 거칠 때마다 특징 맵이 작아지는데, 이를 방지하려면 P=(F−1)/2=1의 '동일(same) 패딩'을 적용해 5×5를 유지한다.

오답 분석

보기①

보기②

정답. (5−3)/1+1 = 3 → 3×3.

보기③

4×4는 필터를 2×2로 계산했을 때 나오는 값이다.

보기④

5×5는 'same 패딩'(P=1)을 적용했을 때의 크기로, 문제는 패딩 미적용이다.

관련 이론 정리

CNN은 합성곱(convolution) 층으로 국소 영역의 특징을 추출하고 풀링(pooling) 층으로 크기를 줄이며 위치 변화에 강인해진다. 합성곱의 핵심 하이퍼파라미터는 필터 크기(F), 스트라이드(S)(필터 이동 간격), 패딩(P)(가장자리 0 채우기)이며, 출력 크기는 (N−F+2P)/S+1로 계산된다. 스트라이드를 키우면 출력이 작아지고 연산량이 줄며, 패딩을 쓰면 크기를 유지하고 가장자리 정보 손실을 막는다. 필터 수만큼 특징 맵 채널이 생성된다. 풀링(최대·평균)은 보통 2×2, 스트라이드 2로 크기를 절반으로 줄인다. 감리에서는 모델 구조의 파라미터 수·연산량(FLOPs)이 목표 환경(엣지·서버)에 적합한지 검토한다.

출력 크기 계산 예:

N F P S 출력
5 3 0 1 3
5 3 1 1 5(same)
5 3 0 2 2
실무 적용 사례

영상 인식 모델 감리 시 계층별 출력 크기·파라미터 수 산정이 타당한지, 엣지 디바이스의 연산·메모리 제약을 충족하는지 검토한다.

감리사 시험 출제 포인트

(N−F+2P)/S+1 공식을 암기하면 즉시 해결된다. 패딩 적용 여부(5×5 유지)와 미적용(3×3)을 구분하는 것이 함정이다.

정답 해설

오버레이 네트워크는 기존 물리망(언더레이) 위에 터널링으로 가상 네트워크를 구성하는 기술이다. VXLAN(UDP 캡슐화), NVGRE(GRE 캡슐화), NVO3(IETF의 오버레이 표준화 작업그룹·프레임워크)는 모두 이 범주에 속한다.

반면 CDPI(Control to Data-Plane Interface)는 SDN 컨트롤러와 데이터평면 장비를 잇는 남향(southbound) 인터페이스 규격(대표적으로 OpenFlow)으로, 오버레이 구현 기술이 아니라 제어 인터페이스다. 따라서 가장 적절하지 않은 것은 ④번이다.

오답 분석

보기①

VXLAN은 L2 프레임을 UDP로 캡슐화하는 대표적 오버레이 기술이다.

보기②

NVGRE는 GRE 터널로 네트워크를 가상화하는 오버레이 기술이다.

보기③

NVO3는 IP 기반 오버레이 네트워크 가상화 표준 프레임워크다.

보기④

정답(부적절). CDPI는 컨트롤러-장비 간 남향 인터페이스 규격으로 오버레이 기술이 아니다.

관련 이론 정리

데이터센터 네트워크 가상화는 크게 오버레이 방식과 SDN 제어 방식으로 접근한다. 오버레이는 물리 토폴로지를 그대로 두고 터널로 논리 네트워크를 만들어, VLAN의 4096개 한계를 넘어 대규모 멀티테넌시를 지원한다(VXLAN은 24비트 VNI로 약 1600만 개 세그먼트). 캡슐화로 오버헤드가 생기고 MTU 조정이 필요하며, 문제 발생 시 가시성 확보가 어렵다. SDN은 제어평면을 분리해 컨트롤러가 전체를 프로그래밍하며, 컨트롤러와 장비 사이는 남향 API(CDPI, 예: OpenFlow), 컨트롤러와 애플리케이션 사이는 북향 API로 연결된다. 실무에서는 SDN 컨트롤러가 VXLAN 오버레이를 오케스트레이션하는 형태로 결합된다. 감리에서는 오버레이 MTU·성능, 컨트롤러 가용성을 점검한다.

오버레이(터널링) VXLAN, NVGRE, NVO3, STT
SDN 남향 인터페이스 CDPI(OpenFlow)
SDN 북향 인터페이스 REST API 등
실무 적용 사례

데이터센터·클라우드 네트워크 감리 시 VXLAN 오버레이의 MTU 설정·성능 오버헤드, 컨트롤러 이중화와 가시성 확보 방안을 점검한다.

오버레이 터널링 기술(VXLAN·NVGRE·NVO3)과 SDN 인터페이스 규격(CDPI/OpenFlow)의 층위를 구분한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

머하웃(Mahout)은 하둡 기반의 확장 가능한 기계학습 라이브러리로, 군집화·분류·추천 알고리즘을 제공하는 분석 레이어 기술이다. 데이터를 모아 전달하는 수집 레이어와는 역할이 다르므로 가장 적절하지 않은 것은 ①번이다.

②스톰은 실시간 스트림 처리 엔진으로 수집·처리 경계에서 스트리밍 데이터를 다루고, ③카프카는 대용량 메시지를 버퍼링·전달하는 분산 메시징 시스템, ④플럼은 로그·이벤트 데이터를 수집해 저장소로 전달하는 도구로 모두 수집 계열에 해당한다.

보기②

관련 이론 정리

빅데이터 아키텍처는 역할별 레이어로 구성된다. 수집(플럼·카프카·스쿱·스톰)은 다양한 원천에서 데이터를 모아 전달하고, 적재(HDFS·NoSQL·HBase)는 저장하며, 처리(MapReduce·Spark·Tez)는 변환·집계, 탐색(Hive·Pig·Impala)은 SQL 등으로 조회, 분석(Mahout·MLlib·R)은 통계·기계학습, 응용(시각화·대시보드)은 결과를 제공한다. 도구별 성격을 보면 스쿱은 RDBMS↔HDFS 배치 전송, 플럼은 로그 스트림 수집, 카프카는 발행-구독 기반 버퍼로 생산자·소비자를 분리해 확장성과 내구성을 준다. 최근에는 배치와 스트리밍을 함께 처리하는 람다·카파 아키텍처가 쓰인다. 감리에서는 데이터 파이프라인의 유실 방지·재처리·모니터링 체계를 점검한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

중복제거(deduplication) 백업은 파일·블록 단위로 해시를 계산해 동일한 데이터는 한 번만 저장하고, 나머지는 그 데이터를 가리키는 메타데이터(포인터)로 관리해 저장 공간을 크게 절약하는 방식이다. ④가 이를 정확히 서술하므로 정답은 ④번이다.

①은 틀림: 전체 백업 이후 변경된 모든 데이터를 백업하는 것은 차등 백업이다(증분은 직전 백업 이후 변경분만). ②도 틀림: "최종 전체 백업 또는 최종 증분 백업 이후 변경분"은 증분 백업의 정의로, 차등과 서술이 뒤바뀌었다. ③도 틀림: 합성 백업은 기본(전체) 백업과 후속 증분 백업들을 결합해 새로운 전체 백업을 만드는 방식이다.

오답 분석

보기①

부적절. 전체 백업 이후 변경된 모든 데이터를 백업하는 것은 차등 백업이다.

보기②

부적절. 최종 백업(전체 또는 증분) 이후 변경분만 백업하는 것은 증분 백업이다.

보기③

부적절. 합성 백업은 전체 백업 + 증분 백업들을 통합해 새 전체 백업을 구성한다.

보기④

정답. 중복제거는 동일 부분을 하나만 저장하고 메타데이터로 구조를 관리한다.

관련 이론 정리

백업 방식은 백업 시간·저장 용량·복구 시간의 절충으로 선택한다. 전체(full) 백업은 복구가 가장 단순하고 빠르지만 시간·용량 부담이 크다. 증분(incremental)은 직전 백업(전체 또는 증분) 이후 변경분만 백업해 백업 시간·용량이 최소지만, 복구 시 전체 + 모든 증분을 순서대로 적용해야 해 복구가 가장 복잡·느리다. 차등(differential)은 마지막 전체 백업 이후 변경분을 매번 백업해 시간이 지날수록 백업량이 커지지만, 복구는 전체 + 마지막 차등 두 개만 있으면 되어 간단하다. 합성 백업은 서버에서 전체+증분을 병합해 가상의 전체 백업을 만들어 운영 부하 없이 복구를 단순화한다. 중복제거는 저장 효율을 높이지만 해시 연산 부하가 있다. 감리에서는 RTO·RPO 목표에 맞는 백업 전략과 복구 테스트 실시 여부를 점검한다.

백업 방식 비교:

방식 대상 복구
전체 전부 가장 간단
증분 직전 백업 이후 변경분 전체+모든 증분(복잡)
차등 마지막 전체 이후 변경분 전체+마지막 차등
중복제거 중복 블록 제거 저장 공간 효율
실무 적용 사례

백업·복구 체계 감리 시 RTO/RPO에 맞는 백업 주기·방식 선택, 복구 테스트 정기 수행, 중복제거·소산 백업 적용을 점검한다.

감리사 시험 출제 포인트

증분(직전 백업 기준)과 차등(마지막 전체 백업 기준)을 바꿔 서술하는 것이 가장 전형적인 함정이다. 두 정의를 확실히 구분한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

라우트 포이즈닝(Route Poisoning)은 경로에 장애가 발생했을 때 해당 경로의 메트릭(홉 수)을 RIP의 무한대 값인 16으로 설정해 광고함으로써, 이웃 라우터들에게 그 경로가 도달 불가능함을 명확하고 신속하게 알리는 기법이다. 문제 서술과 정확히 일치하므로 정답은 ②번이다.

①Hold Down은 장애 경로에 대해 일정 시간 동안 갱신 정보를 받아들이지 않아 잘못된 경로가 되살아나는 것을 막는 기법, ③Split Horizon은 경로를 배운 인터페이스로는 그 경로를 다시 광고하지 않는 기법, ④Triggered Update는 주기를 기다리지 않고 변경 발생 즉시 갱신을 전송하는 기법이다.

오답 분석

보기①

Hold Down은 일정 시간 해당 경로의 갱신을 무시해 안정화하는 기법이다.

보기②

정답. 포이즈닝은 홉 16(무한대)으로 광고해 도달 불가를 알린다.

보기③

Split Horizon은 배운 방향으로 되돌려 광고하지 않아 루프를 막는다.

보기④

Triggered Update는 변화 즉시 전송으로 수렴을 앞당기는 기법이다.

관련 이론 정리

거리벡터 라우팅은 이웃의 정보를 그대로 신뢰하므로 장애 시 잘못된 경로가 서로 순환 참조되어 홉 수가 무한히 증가하는 카운트 투 인피니티(count to infinity) 문제가 생긴다. RIP는 최대 홉을 15로 제한하고 16을 무한대(도달 불가)로 정의해 이 문제를 유한 시간에 종료시킨다. 완화 기법으로 ① 스플릿 호라이즌(배운 인터페이스로 재광고 금지), ② 포이즌 리버스(배운 인터페이스로 홉 16으로 광고해 명시적으로 무효화), ③ 라우트 포이즈닝(장애 경로를 16으로 광고), ④ 홀드다운 타이머(불안정 경로를 일정 시간 고정), ⑤ 트리거드 업데이트(즉시 전파)를 함께 사용한다. 링크상태 프로토콜(OSPF)은 전체 토폴로지를 공유해 이런 문제가 근본적으로 적다. 감리에서는 라우팅 수렴 시간과 루프 방지 설정을 점검한다.

정답 해설

OpenFlow는 SDN 컨트롤러가 스위치의 전달 동작을 제어하기 위한 남향 인터페이스 규격으로, 주요 구성요소는 플로우 테이블(매치 조건-액션 규칙 저장), 그룹 테이블(다중 경로·멀티캐스트 등 그룹 액션 처리), 파이프라인 처리(여러 플로우 테이블을 순차 통과시키며 매칭), 그리고 컨트롤러와의 보안 채널이다.

③의 NFV(Network Function Virtualization)는 방화벽·로드밸런서 같은 네트워크 기능을 전용 장비에서 분리해 범용 서버의 소프트웨어로 구현하는 별개의 개념으로, OpenFlow의 구성요소가 아니다. 따라서 정답은 ③번이다.

보기④

관련 이론 정리

OpenFlow 스위치는 패킷이 들어오면 플로우 테이블의 매치 필드(입력 포트, MAC/IP 주소, 포트 번호 등)와 대조해 일치하는 규칙의 액션(전달·폐기·수정·컨트롤러 전송)을 수행한다. 여러 테이블을 두고 순차적으로 처리하는 파이프라인 구조는 복잡한 정책을 단계적으로 표현하게 해준다. 매칭되는 규칙이 없으면 컨트롤러에 문의(Packet-In)하고 컨트롤러가 규칙을 내려준다(Flow-Mod). 한편 SDN과 NFV는 상호 보완적이지만 별개다: SDN은 제어평면 분리로 네트워크를 프로그래밍하고, NFV는 네트워크 기능(장비)의 소프트웨어화를 지향한다. 둘을 결합하면 서비스 체이닝을 유연하게 구성할 수 있다. 감리에서는 SDN/NFV 도입 시 성능·가용성·보안(컨트롤러 보호)을 점검한다.

SDN vs NFV:

구분 초점
SDN(OpenFlow) 제어평면 분리·전달 제어
NFV 네트워크 기능의 소프트웨어화
실무 적용 사례

SDN/NFV 도입 감리 시 플로우 테이블 용량·규칙 수 한계, 컨트롤러 장애 시 동작, 가상 네트워크 기능의 성능(패킷 처리량)을 점검한다.

감리사 시험 출제 포인트

OpenFlow 구성요소(플로우·그룹 테이블, 파이프라인)와 별개 개념인 NFV를 구분한다. SDN과 NFV의 차이도 자주 출제된다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

①의 FDDI, PPP, SLIP, ATM은 모두 데이터링크(2계층) 계열 프로토콜로 올바르게 짝지어졌다. FDDI는 광 토큰링 LAN, PPP·SLIP은 점대점 링크 프로토콜, ATM은 셀 기반 전송 기술로 링크 계층에서 동작한다. 따라서 정답은 ①번이다.

③은 틀림: ARP는 전송계층(4계층)이 아니라 데이터링크/네트워크 경계에서 동작하는 주소 변환 프로토콜이다. ④도 틀림: SPX는 전송계층(4계층) 프로토콜로 세션 계층 목록에 포함될 수 없다. ②의 DDP(AppleTalk Datagram Delivery Protocol)도 목록 구성상 다른 보기들과 계층 일관성 논란이 있어, 가장 올바르게 짝지어진 것은 ①이다.

보기④

SPX는 4계층(전송) 프로토콜로 세션 계층 목록에 포함될 수 없다.

관련 이론 정리

OSI 계층별 대표 프로토콜을 정리하면, 1계층은 전기·광 신호 규격(RS-232, 이더넷 물리 규격), 2계층은 이더넷, PPP·SLIP(점대점), HDLC, FDDI, ATM, 프레임릴레이, 3계층은 IP, ICMP, IGMP, 라우팅 프로토콜(OSPF·RIP), 4계층은 TCP, UDP, SPX, SCTP, 5계층은 세션 관리(NetBIOS, RPC, Named pipe), 6계층은 표현(암호화·압축·인코딩), 7계층은 HTTP·FTP·SMTP·DNS·SNMP다. ARP는 IP 주소를 MAC으로 변환해 2·3계층 경계에 걸쳐 있는 것으로 분류되며 전송계층은 아니다. 프로토콜을 계층으로 묻는 문제는 경계에 있는 프로토콜(ARP·ICMP)과 이름이 비슷한 것(SPX/IPX)에서 오답이 만들어진다. 감리에서는 프로토콜 스택 설계와 계층별 보안 통제를 점검한다.

정답 해설

L3 스위치는 3계층의 IP 주소를 보고 스위칭(라우팅)하며, 하드웨어(ASIC) 기반이라 라우터보다 빠르지만 라우터가 WAN 인터페이스·다양한 프로토콜·NAT·VPN 등 훨씬 다양한 기능을 수행한다. ②가 이를 정확히 서술하므로 정답은 ②번이다.

①은 틀림: L2 스위치는 MAC 주소만 보고 판단하며 IP를 보고 라우터로 보낼지 결정하는 것은 3계층 기능이다. ③도 틀림: L4 스위치의 부하분산 서술 자체는 맞으나 "IP 주소를 기반으로 스위칭"이라는 핵심 규정이 부정확하다(L4는 포트 번호 등 전송계층 정보로 스위칭). ④도 틀림: L7 스위치는 오히려 URL·콘텐츠 기반의 정교한 부하분산을 수행하므로 "부하분산 기능은 없다"는 명백한 오류다.

보기③

부적절. L4는 전송계층(포트) 정보 기반이며 IP 기반이라는 규정이 부정확하다.

보기④

관련 이론 정리

스위치는 참조하는 계층 정보에 따라 기능이 확장된다. L2 스위치는 MAC 주소 테이블로 프레임을 전달하며 충돌 도메인을 분할한다. L3 스위치는 IP 라우팅을 하드웨어로 처리해 대규모 내부망 라우팅에 쓰이며, 라우터 대비 처리 속도가 빠르지만 WAN 프로토콜·고급 기능은 제한적이다. L4 스위치는 TCP/UDP 포트 번호와 세션 정보를 보고 여러 서버로 트래픽을 분산(로드밸런싱)하며 헬스체크로 장애 서버를 제외한다. L7 스위치는 HTTP URL·쿠키·헤더 등 애플리케이션 콘텐츠를 해석해 URL별 서버 분기, SSL 오프로딩, 캐싱 같은 정교한 처리를 수행한다(현대의 ADC·WAF와 결합). 감리에서는 부하분산 구성·헬스체크·세션 유지(스티키) 설정과 이중화를 점검한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

마이크로서비스는 개별 서비스를 독립적으로 배포할 수 있다는 점은 맞지만, 다수의 서비스가 네트워크로 통신하는 분산 시스템이므로 서비스 간 상호작용을 검증하는 통합·종단(E2E) 테스트가 모놀리틱보다 훨씬 어렵다. 서비스 조합·버전·네트워크 장애 시나리오를 모두 검증해야 하고, 테스트 환경 구성도 복잡하다.

④는 "모놀리틱에 비해 테스트가 쉽다"고 서술했으므로 부적절하다. 따라서 정답은 ④번이다(개별 단위 테스트는 쉬울 수 있으나 애플리케이션 전체 테스트는 더 어렵다). ①②③은 MSA의 일반적 장점 서술로 옳다.

오답 분석

보기①

옳음. 거대한 단일 코드베이스에서 벗어나 소규모 팀이 작은 서비스를 담당한다.

보기②

옳음. 비즈니스 기능 중심 설계와 분산 제어, 독립적 자동 배포가 MSA의 특징이다.

보기③

옳음. 모놀리틱이 달성하기 어려운 모듈화를 추구해 개별 서비스 구현·이해가 용이하다.

보기④

정답(부적절). 분산 구조로 인해 전체 애플리케이션 테스트는 모놀리틱보다 어렵다.

관련 이론 정리

MSA는 애플리케이션을 비즈니스 기능 단위의 작은 서비스로 나누고, 각 서비스가 독립된 프로세스·데이터 저장소를 가지며 API로 통신하는 아키텍처다. 장점은 독립 배포·기술 스택 자율성·부분 확장·장애 격리·팀 자율성이다. 반면 단점(대가)이 분명한데, 분산 시스템의 복잡성(네트워크 지연·부분 실패), 데이터 일관성 확보 어려움(분산 트랜잭션 대신 Saga·이벤트 기반 최종 일관성), 통합 테스트·디버깅·모니터링의 난도 상승(분산 추적 필요), 운영 도구(컨테이너·오케스트레이션·서비스 메시) 요구가 그것이다. 그래서 조직 규모와 도메인 복잡도가 충분치 않으면 모놀리식이 더 적합할 수 있다. 감리에서는 서비스 분해 기준, 데이터 일관성 전략, 관측성(로그·추적) 체계를 점검한다.

감리사 시험 출제 포인트

MSA는 배포는 쉬워지지만 테스트·운영은 어려워진다는 트레이드오프가 핵심이다. 장점만 나열한 서술 중 과장된 것을 찾는 문제다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

Type1(베어메탈) 하이퍼바이저는 하드웨어 위에 직접 설치되어 하드웨어를 제어하므로 호스트 OS를 거치지 않아 오버헤드가 적고 성능이 우수하다(예: VMware ESXi, Xen, Hyper-V). 반면 Type2(호스티드)는 일반 OS 위에 응용 프로그램처럼 설치되어 호스트 OS를 한 단계 더 거치므로 오버헤드가 크다(예: VirtualBox, VMware Workstation, Virtual PC).

③은 "Type1이 Type2보다 오버헤드가 있지만 하이퍼바이저가 가볍다"고 서술했는데, 오버헤드 관계가 반대이므로 틀린 설명이다. 따라서 정답은 ③번이다. ①(반가상화에서 하이퍼바이저가 특권모드 차지), ②(Type1은 하드웨어 직접 제어), ④(VirtualBox·Virtual PC는 Type2)는 모두 옳다.

오답 분석

보기①

옳음. 가상화에서 하이퍼바이저가 특권(커널) 모드를 차지하고 게스트는 낮은 권한에서 동작한다.

보기②

옳음. Type1은 하드웨어 위에 직접 올라가 하드웨어를 제어한다.

보기③

정답(부적절). Type1이 Type2보다 오버헤드가 적다(성능 우수).

보기④

옳음. VirtualBox·Virtual PC는 호스트 OS 위에서 동작하는 Type2다.

관련 이론 정리

하이퍼바이저는 물리 자원을 여러 가상머신에 분배·격리하는 소프트웨어다. Type1(네이티브·베어메탈)은 하드웨어에 직접 설치돼 자원 제어가 직접적이라 성능·보안이 우수하고 서버·데이터센터 환경에 쓰인다. Type2(호스티드)는 기존 OS 위에서 애플리케이션처럼 실행돼 설치·사용이 간편하고 개발·테스트·데스크톱 환경에 적합하지만, 호스트 OS 계층으로 인한 성능 손실과 호스트 OS 취약점 노출이 단점이다. 가상화 구현 방식(전가상화/반가상화)과 하이퍼바이저 유형(Type1/Type2)은 서로 다른 분류 축이므로 혼동하지 않아야 한다. 하드웨어 지원 가상화(Intel VT-x, AMD-V)는 특권 명령 처리를 CPU가 도와 성능을 높인다. 감리에서는 워크로드 성격에 맞는 하이퍼바이저 선택과 자원 과할당·격리 보안을 점검한다.

실무 적용 사례

가상화 인프라 감리 시 운영 서버에 Type1을 적용했는지, 자원 과할당(오버커밋)으로 성능 저하가 없는지, 하이퍼바이저 보안 패치가 관리되는지 점검한다.

감리사 시험 출제 포인트

Type1이 오버헤드가 적고 성능이 우수하다는 점이 핵심이다. 오버헤드 관계를 뒤집는 것이 전형적 함정이며, 제품명(ESXi=Type1, VirtualBox=Type2)도 함께 암기한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

무결성(integrity)은 데이터가 인가된 방식으로만 변경되어야 한다는 원칙이다. 그러나 무결성 침해는 악의적 공격뿐 아니라 사용자 실수, 소프트웨어 버그, 하드웨어 결함, 전송 오류, 자연재해 등 비악의적 원인으로도 발생한다.

④는 "무결성 침해는 악의적인 행동의 결과로만 발생한다"고 단정했으므로 틀린 설명이다. 따라서 정답은 ④번이다. ①(대칭키=동일 키), ②(DoS=가용성 위협), ③(스테가노그래피=감춰진 기록)은 모두 옳다.

오답 분석

보기①

옳음. 대칭 키 암호는 암·복호화에 같은 키를 사용한다(AES·DES).

보기②

옳음. DoS/DDoS는 서비스 이용을 방해해 가용성을 위협한다.

보기③

옳음. 암호(cryptography)는 내용을 숨기고, 스테가노그래피는 존재 자체를 숨긴다.

보기④

정답(부적절). 무결성 침해는 실수·오류 등 비악의적 원인으로도 발생한다.

관련 이론 정리

정보보호의 3대 목표는 기밀성(Confidentiality)(인가된 자만 접근), 무결성(Integrity)(인가된 방식으로만 변경), 가용성(Availability)(필요할 때 사용 가능)의 CIA 삼각형이다. 각 목표를 위협하는 대표 공격은 기밀성-도청·스니핑, 무결성-변조·위조, 가용성-DoS/DDoS다. 무결성 보호 수단으로는 해시(SHA-256)·MAC·전자서명·접근통제·버전 관리가 있으며, 전송 오류로 인한 무결성 훼손은 CRC·체크섬으로도 검출한다. 중요한 관점은 보안 사고의 원인이 외부 공격만이 아니라 내부자 실수와 시스템 결함이 상당 비중을 차지한다는 점으로, 따라서 통제는 예방·탐지·복구를 아우르고 백업·변경관리 절차를 포함해야 한다. 스테가노그래피는 이미지·음성에 데이터를 숨기는 은닉 기술로 워터마킹과 함께 저작권 보호에 쓰이나 악성코드 은닉에도 악용된다. 감리에서는 CIA 관점의 통제 적정성을 점검한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

앙상블 학습은 여러 개의 개별 학습기(약한 학습기)를 결합해 단일 모델보다 높은 성능과 일반화를 얻는 기법이다. 문제의 "여러 분류기를 하나의 메타 분류기로 연결"이라는 서술과 정확히 일치하므로 정답은 ②번이다.

①다변량 선형회귀는 단일 회귀 모델, ③K-means는 비지도 군집화 알고리즘, ④LSTM은 시퀀스 처리를 위한 순환신경망 구조로, 모두 여러 분류기를 결합하는 기법이 아니다.

오답 분석

보기①

다변량 선형회귀는 여러 독립변수를 사용하는 단일 회귀 모델이다.

보기②

정답. 앙상블 학습은 여러 분류기를 결합해 성능을 높인다.

보기③

K-means는 레이블 없이 군집을 찾는 비지도 학습이다.

보기④

LSTM은 장기 의존성을 학습하는 순환신경망 구조다.

관련 이론 정리

앙상블은 결합 방식에 따라 세 갈래로 나뉜다. 배깅(Bagging)은 원본 데이터를 부트스트랩 샘플링해 여러 모델을 병렬 학습시키고 투표·평균으로 결합해 분산(과적합)을 줄인다(대표: 랜덤 포레스트). 부스팅(Boosting)은 이전 모델이 틀린 샘플에 가중치를 주어 순차적으로 보완 학습해 편향을 줄인다(대표: AdaBoost, Gradient Boosting, XGBoost, LightGBM). 스태킹(Stacking)은 여러 기반 모델의 예측을 입력으로 받는 메타 모델을 학습시켜 결합한다. 앙상블은 대체로 단일 모델보다 성능이 우수해 정형 데이터 분석·대회에서 표준처럼 쓰이지만, 모델이 여러 개라 학습·추론 비용이 크고 해석이 어렵다. 감리에서는 모델 복잡도 대비 성능 이득, 설명 가능성 요구(금융·의료)와의 균형을 점검한다.

기법 방식 효과 예

부스팅 순차·가중 편향↓ XGBoost
스태킹 메타 모델 결합 성능↑ -
실무 적용 사례

예측 모델 감리 시 앙상블 적용의 성능 이득과 추론 비용·설명 가능성 요구(규제 산업)의 균형이 적절한지 검토한다.

정답 해설

IPv4→IPv6 전환 기술은 세 가지로 구분된다.

  • 이중 스택(Dual Stack): 하나의 장비가 IPv4와 IPv6 프로토콜 스택을 모두 탑재해 상대에 따라 골라 쓰는 방식
  • 터널링(Tunneling): IPv6 패킷을 IPv4 패킷 안에 캡슐화해 IPv4 망을 통과시키는 방식(6to4, ISATAP 등)
  • 주소 변환(Translation): IPv4와 IPv6 헤더를 상호 변환(NAT64 등)

④는 이중 스택을 설명하면서 터널링의 정의(IPv6를 IPv4에 캡슐화)를 서술했으므로 틀렸다. 따라서 정답은 ④번이다. ①(128비트), ②(실시간 멀티미디어 지원 - 흐름 레이블·QoS), ③(다양한 망에서 효율적 동작)은 모두 옳다.

관련 이론 정리

IPv6는 IPv4의 주소 고갈을 근본적으로 해결하기 위해 128비트 주소를 도입했고, 40바이트 고정 헤더와 확장 헤더 구조로 라우터 처리를 단순화했다. 브로드캐스트를 없애고 애니캐스트를 도입했으며, 주소 자동설정(SLAAC), IPsec 통합 지원, 흐름 레이블을 통한 QoS 지원이 특징이다. 전면 전환이 불가능하므로 과도기에는 이중 스택(가장 일반적, 두 스택 병행 운영), 터널링(IPv6 섬을 IPv4 바다로 연결), 변환(NAT64/DNS64)을 병행한다. 이중 스택은 구현이 단순하지만 두 체계를 모두 관리해야 하고 IPv4 주소가 여전히 필요하며, 터널링은 캡슐화 오버헤드와 MTU·보안 이슈가 있다. 감리에서는 IPv6 전환 계획, 듀얼스택 구간의 방화벽 정책이 IPv4와 동등하게 적용됐는지 점검한다.

기술 방식

변환(NAT64) 헤더 상호 변환
실무 적용 사례

IPv6 전환 감리 시 듀얼스택 구간의 IPv6 보안정책 누락(방화벽·IDS 미적용) 여부, 터널 구간 MTU·성능, 단계적 전환 계획의 실효성을 점검한다.

감리사 시험 출제 포인트

이중 스택(둘 다 지원)과 터널링(캡슐화)의 정의를 바꿔 놓는 것이 가장 전형적인 함정이다. IPv6 128비트·40바이트 헤더도 함께 암기한다.
