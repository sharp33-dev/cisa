영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

「정보시스템 하드웨어 규모 산정 지침」의 연계 부하 보정은 WAS 서버에 대해 연계 업무의 성격에 따라 추가 보정치를 적용하는 규정이다. WEB 서버에는 연계 부하 보정을 적용하지 않으며(①), 연계 업무가 없는 WAS에도 보정치를 적용하지 않는다(②). 갱신성 연계 업무가 있는 경우 상대적으로 높은 보정치(30%)를 적용한다(④).

③은 "단순 조회 업무만 있는 경우에 전체 부하의 20%를 추가 적용한다"고 서술했으나, 지침상 단순 조회성 연계 업무의 보정치는 갱신성보다 낮게 규정되며 문항이 제시한 수치·조건과 일치하지 않아 가장 적절하지 않다. 따라서 정답은 ③번이다.

오답 분석

보기①

옳음. WEB 서버는 연계 부하 보정 대상이 아니다.

보기②

옳음. 연계 업무가 없는 WAS는 추가 보정치를 적용하지 않는다.

보기③

정답(부적절). 단순 조회성 연계 업무의 보정 조건·수치가 지침과 어긋난다.

보기④

옳음. 대내/대외 갱신 연계 업무가 있으면 더 높은 보정치(30%)를 적용한다.

관련 이론 정리

정보시스템 하드웨어 규모 산정 지침(TTAK.KO-10.0292)은 서버 유형(WEB/WAS/DB 등)별로 기준 처리 용량을 산출한 뒤 여러 보정 계수를 곱해 실제 규모를 결정한다. 대표 보정 항목은 ① 연계 부하 보정(타 시스템과의 연계로 인한 추가 부하), ② 피크(최대 동시 사용) 부하 보정, ③ 여유율(버퍼) 보정이다. 이 중 연계 부하 보정은 WAS 서버에만 적용하고 WEB 서버에는 적용하지 않으며, 연계 업무의 성격이 단순 조회성인지 갱신성(대내·대외)인지에 따라 보정치가 차등 적용된다. 갱신성 연계는 트랜잭션·정합성 부담이 커 더 높은 보정치를, 연계 업무가 없으면 보정을 적용하지 않는다. 감리에서는 이런 보정 근거가 산정서에 정량적으로 명시됐는지 확인한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

「인공지능 학습용 데이터 품질관리 가이드라인」에서 AI 데이터 품질 지표는 공정 단계(준비·구축·결과 등)별로 구분된다. 구축공정 품질 지표로는 완전성, 유용성, 준비성 등이 제시된다.

③의 유효성은 구축공정이 아니라 다른 공정(결과/활용 관점)의 지표에 해당하여 구축공정 품질 지표로는 가장 적절하지 않다. 따라서 정답은 ③번이다.

오답 분석

보기①

완전성은 구축공정에서 데이터가 빠짐없이 구축됐는지 보는 지표다.

보기②

유용성은 구축된 데이터가 학습에 유용한지 보는 지표다.

보기③

정답(부적절). 유효성은 구축공정 품질 지표로 제시되는 항목이 아니다.

보기④

준비성은 구축을 위한 준비 상태를 보는 지표로 구축공정 관련 항목이다.

관련 이론 정리

인공지능 학습용 데이터의 품질은 데이터 생명주기 공정(계획·수집·준비·구축·결과·활용)에 걸쳐 단계별 품질 지표로 관리한다. 「AI 학습용 데이터 품질관리 가이드라인」은 각 공정에서 확인해야 할 품질 특성을 정의하는데, 구축공정에서는 데이터가 요구 규격대로 빠짐없이 만들어졌는지(완전성), 학습에 실제로 쓸모가 있는지(유용성), 구축을 위한 준비가 갖춰졌는지(준비성) 등을 본다. 반면 정확성·유효성처럼 결과·활용 관점의 지표는 다른 공정에서 다룬다. 초거대 AI에서는 데이터 규모·다양성·대표성·편향 관리가 특히 중요하며, 감리에서는 공정별 지표가 측정·기록·개선되는 체계가 있는지 점검한다.

AI 데이터 품질 공정별 지표(예):

공정 지표
구축 완전성·유용성·준비성 등
결과/활용 유효성·정확성 등
공통 다양성·대표성
실무 적용 사례

AI 학습데이터 구축 사업 감리 시 공정 단계별 품질 지표(완전성·정확성·유효성 등)를 가이드라인에 맞게 측정·관리하는지 점검한다.

감리사 시험 출제 포인트

AI 데이터 품질 가이드라인의 공정별 지표 소속을 구분한다. 유효성(결과공정)을 구축공정 지표로 오인하지 않도록 한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

부동 소수점 덧셈의 절차는 다음과 같다. (1) 지수 비교·조정(정렬)으로 두 수의 지수를 맞추고, (2) 가수(mantissa) 덧셈을 수행한 뒤, (3) 결과 정규화와 반올림을 한다. 즉 지수 조정(②)·가수 덧셈(④)·정규화(①)가 필요하다.

③ 지수 덧셈은 두 수의 지수를 서로 더하는 연산으로, 이는 부동 소수점 곱셈에서 필요한 과정이지 덧셈에서는 사용하지 않는다. 따라서 덧셈 과정에서 필요하지 않은 연산은 ③번이다.

오답 분석

보기①

정규화는 덧셈 결과를 표준 형식으로 맞추는 필수 단계다.

보기②

지수 조정(정렬)은 두 수의 지수를 맞추는 덧셈 전처리 단계다.

보기③

정답(부적절). 지수 덧셈은 곱셈에서 지수를 더하는 연산으로, 덧셈에는 필요 없다.

보기④

가수 덧셈은 지수를 맞춘 뒤 실제 값을 더하는 핵심 단계다.

관련 이론 정리

부동소수점 수는 부호·지수·가수(mantissa)로 구성되며, 연산 종류에 따라 처리 절차가 다르다. 덧셈/뺄셈은 두 수의 소수점 위치를 맞추기 위해 먼저 지수를 비교해 작은 쪽을 큰 쪽에 맞추는 지수 정렬(조정)을 하고, 그다음 가수를 더한 뒤, 결과를 표준 형식으로 만드는 정규화와 반올림을 수행한다. 곱셈은 성격이 달라 지수는 서로 더하고(지수 덧셈) 가수는 서로 곱한 뒤 정규화한다. 즉 '지수 덧셈'은 곱셈 고유의 단계이지 덧셈에는 등장하지 않는다. 이런 절차 차이 때문에 자릿수 손실·반올림 오차가 발생할 수 있어, 수치 정확도가 중요한 시스템에서는 연산 순서와 정밀도 관리를 함께 검토한다.

가수 더하기 곱하기

실무 적용 사례

수치 연산 정확도 감리 시 부동소수점 연산의 정규화·반올림·자릿수 손실을 이해해 계산 오차·정밀도 문제를 진단한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

F=1인 경우는 A=0인 모든 행(000,001,010,011)과 111이다.

  • A=0인 4개 행이 모두 1이므로 A′로 묶인다.
  • A=1에서 F=1인 경우는 111(ABC)뿐이다.

따라서 F = A′ + ABC이고, 흡수 법칙(A′ + ABC = A′ + BC)을 적용하면 F = A′ + BC로 최소화된다. 정답은 ①번이다.

검증: A′ + BC가 111(BC=1)을 포함하고, A=1의 나머지(100,101,110)는 A′도 BC도 아니어서 0이 되어 진리표와 일치한다.

오답 분석

보기①

정답. A′로 A=0 전체를, BC로 111을 커버 → 최소식.

보기②·③

보기④

A′B′C′, A′B′C 등 개별 최소항으로 전개한 것으로, A′로 묶으면 되는 것을 세분화한 비최소 형태다.

관련 이론 정리

부울 함수 최소화는 진리표나 논리식을 리터럴·항의 수가 가장 적은 등가식으로 줄여 게이트 수·지연·비용을 낮추는 작업이다. 대표 기법인 카르노맵(K-map)은 인접한(1비트만 다른) 1들을 2의 거듭제곱 크기(1·2·4·8…)의 그룹으로 최대한 크게 묶어, 각 그룹이 하나의 곱항(product term)이 되도록 한다. 이때 흡수 법칙(A′+A′BC=A′, A′+ABC=A′+BC 등)과 상보·분배 법칙이 반복적으로 적용된다. 변수 하나가 0/1 전 영역에서 출력이 같다면 그 변수는 소거된다(예: A=0 전체가 1이면 A′로 묶임). 최소화 결과는 반드시 원래 진리표와 동치인지 재검증해야 하며, 회로 설계·검증 감리에서 게이트 최적화의 정당성을 확인하는 근거가 된다.

정답 해설

각 업무의 서비스 시간과 이용률(가동률)을 구한다.

관련 이론 정리

시스템 처리량(throughput)은 대기행렬 이론의 이용률(utilization) 개념으로 산정한다. 한 자원의 이용률 ρ는 '도착률 × 서비스시간'이며, 서비스시간은 '1 / 개별 최대 처리율(TPS)'이다. 여러 업무가 같은 자원을 공유하면 각 업무의 이용률을 합산하고, 자원이 포화(이용률 100%)되는 지점이 곧 시스템 최대 처리량이다. 따라서 최대 처리량 = 현재 총 도착률 / 현재 총 이용률로 계산된다. 이용률이 1에 가까워질수록 대기시간이 급격히 늘어나므로(응답시간 곡선의 무릎), 실제 용량 설계는 목표 이용률(예: 70~80%) 이하로 여유를 둔다. 감리에서는 업무별 서비스시간·도착률 근거와 병목 자원 식별이 타당한지 확인한다.

RAID 1+0(RAID 10)은 미러링(RAID 1) + 스트라이핑(RAID 0) 결합으로, 미러 쌍을 만들고 이를 스트라이핑한다.
- 가: 미러 쌍(2개) × 스트라이핑(2조) = 최소 4개 디스크 필요 → 옳음.
- 나: RAID 1+0은 미러링(복제)으로 복구하며 패리티를 사용하지 않는다(패리티는 RAID 5/6). → 틀림.
- 다: 미러 복제본이 있으므로 임의의 디스크 하나가 고장나도 정상 운영된다 → 옳음.

따라서 옳은 것은 가·다이며 정답은 ②번이다.

오답 분석

보기①·③·④

'나'(패리티 디스크 활용)는 RAID 1+0의 특성이 아니다(패리티는 RAID 5/6). 따라서 '나'를 포함한 보기는 옳지 않다.

보기②

정답. 가(최소 4개)·다(단일 장애 허용)가 옳다.

관련 이론 정리

RAID(Redundant Array of Independent Disks)는 여러 물리 디스크를 묶어 성능과 가용성을 높이는 기술로, 스트라이핑(분산 저장으로 성능↑)·미러링(복제로 가용성↑)·패리티(연산으로 복구 정보 생성)의 조합으로 레벨이 나뉜다. RAID 0은 스트라이핑만 하여 성능은 좋지만 이중화가 없어 한 디스크만 고장나도 전체가 손실된다. RAID 1은 미러링으로 복제본을 두고, RAID 5/6은 패리티를 분산 저장해 디스크 1개(또는 2개) 고장까지 복구한다. RAID 1+0(10)은 미러 쌍을 만든 뒤 이를 스트라이핑한 결합형으로, 최소 4개 디스크가 필요하고 패리티 대신 미러 복제로 복구하므로 쓰기 성능 저하가 적고 단일 디스크 장애를 견딘다. 감리에서는 성능·가용성·비용 요구에 맞는 레벨 선택과 핫스페어·리빌드 정책을 점검한다.

RAID 레벨:

RAID 1 미러링 2
RAID 5 분산 패리티 3
RAID 1+0 미러+스트라이프 4
실무 적용 사례

스토리지 구성 감리 시 성능·가용성 요구에 맞는 RAID 레벨(1+0 vs 5/6)을 선택하고, 최소 디스크 수·장애 허용 범위가 요구를 충족하는지 확인한다.

감리사 시험 출제 포인트

RAID 1+0은 미러링 기반(패리티 미사용)이며 최소 4개 디스크·단일 장애 허용이라는 특성을 구분한다. 패리티(RAID 5/6)와 혼동하는 함정을 배제한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

IPv6 기본 헤더의 필드는 Version, Traffic Class, Flow Label, Payload Length, Next Header, Hop Limit, 출발지/목적지 주소이다.

IPv4의 TTL(Time to Live)은 IPv6에서 Hop Limit으로 이름이 바뀌었다. 즉 IPv6 헤더에는 'Time to Live'라는 필드가 존재하지 않는다. 따라서 IPv6 헤더 필드로 적절하지 않은 것은 ③번 Time to Live이다. (①Hop Limit, ②Flow Label, ④Traffic Class는 모두 IPv6 헤더 필드다.)

오답 분석

보기①

Hop Limit은 IPv6에서 TTL을 대체하는 홉 수 제한 필드다.

보기②

Flow Label은 IPv6에서 특정 흐름을 식별해 QoS를 지원하는 필드다.

보기③

정답(부적절). Time to Live는 IPv4 필드이며, IPv6에서는 Hop Limit으로 대체되어 존재하지 않는다.

보기④

Traffic Class는 IPv6에서 우선순위/서비스 유형을 나타내는 필드다.

관련 이론 정리

IPv6는 IPv4의 주소 고갈과 헤더 복잡성을 개선하기 위해 128비트 주소와 고정 40바이트 기본 헤더를 채택했다. IPv4의 가변 헤더·옵션은 IPv6에서 확장 헤더 체인(Next Header)으로 분리해 기본 헤더를 단순화했고, 라우터 부담이 큰 헤더 체크섬은 아예 제거했다(상위·하위 계층에서 무결성 보장). 필드 대응 관계를 보면 IPv4의 TTL은 IPv6에서 Hop Limit으로, Type of Service는 Traffic Class로 이름이 바뀌었고, 특정 트래픽 흐름을 식별해 QoS를 지원하는 Flow Label이 새로 추가됐다. 즉 'Time to Live'는 IPv4 전용 명칭이며 IPv6 헤더에는 존재하지 않는다. IPv6 전환 감리에서는 듀얼스택·터널링·주소 체계 변경에 따른 장비·정책 호환성을 확인한다.

IPv4 vs IPv6 헤더:

IPv4 IPv6
TTL Hop Limit
Type of Service Traffic Class
(없음) Flow Label
Header Checksum 제거(간소화)
실무 적용 사례

IPv6 전환 감리 시 헤더 필드 변경(TTL→Hop Limit, 체크섬 제거, Flow Label 추가)을 이해해 네트워크 장비·정책 호환성을 검토한다.

감리사 시험 출제 포인트

IPv6의 Hop Limit이 IPv4 TTL을 대체한다는 점을 안다. 'Time to Live'를 IPv6 필드로 오인하는 함정을 배제한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

다단계 피드백 큐(MLFQ)의 핵심 특징은 프로세스가 실행 이력(CPU 사용량)에 따라 큐 사이를 이동(feedback)할 수 있다는 점이다. 타임퀀텀을 다 쓴 프로세스는 낮은 우선순위 큐로 강등되고, 오래 기다린 프로세스는 상위로 승격(aging)될 수 있다.

③은 "처음 배정된 큐에 계속 머무르며 다른 큐로 이동할 수 없다"고 했으나, 이는 다단계 큐(비피드백)의 특성이며 MLFQ는 큐 간 이동이 가능하다. 따라서 ③이 적절하지 않다. 정답은 ③번이다.

①(큐별 다른 알고리즘)·②(큐별 다른 타임퀀텀)·④(우선순위 높은 큐 우선 할당)는 MLFQ의 옳은 특성이다.

오답 분석

보기①

옳음. 각 큐는 RR·FCFS 등 서로 다른 스케줄링을 사용할 수 있다.

보기②

옳음. 큐마다 타임퀀텀을 다르게 설정할 수 있다.

보기③

정답(부적절). MLFQ는 큐 간 이동(피드백)이 가능하다. "이동 불가"는 다단계 큐 특성이다.

보기④

옳음. 상위 우선순위 큐가 비어있지 않으면 그 큐를 우선 할당한다.

관련 이론 정리

다단계 큐(Multi-level Queue)는 프로세스를 우선순위·성격(대화형/배치 등)에 따라 여러 개의 준비 큐로 나누고, 각 큐마다 서로 다른 스케줄링 알고리즘과 타임퀀텀을 둘 수 있게 한 방식이다. 기본형은 한번 배정된 큐에 계속 머물러(고정) 낮은 우선순위 큐가 기아(starvation)에 빠지기 쉽다. 이를 개선한 다단계 피드백 큐(MLFQ)는 프로세스의 실행 이력에 따라 큐 사이를 이동시킨다. 타임퀀텀을 다 쓴(CPU를 오래 점유한) 프로세스는 낮은 우선순위 큐로 강등하고, 오래 기다린 프로세스는 상위 큐로 승격(aging)해 기아를 방지한다. 그 결과 짧은 대화형 작업은 빠른 응답을, 긴 배치 작업은 충분한 처리량을 얻는 균형을 이룬다. 감리에서는 강등·승격 기준과 타임퀀텀 설정이 워크로드 특성에 맞는지 본다.

큐별 알고리즘 가능 가능
타임퀀텀 큐별 큐별
기아 방지 어려움 aging으로 완화
실무 적용 사례

OS 스케줄링 정책 감리 시 대화형·배치 작업의 응답성·처리량 균형을 위해 MLFQ의 큐 강등·승격(aging) 정책이 적절히 설정됐는지 검토한다.

감리사 시험 출제 포인트

MLFQ의 큐 간 이동(피드백) 특성을 다단계 큐(고정)와 구분한다. "큐 이동 불가"라는 함정(③)을 배제한다.

정답 해설

플린(Flynn) 분류에서 SIMD(Single Instruction Multiple Data)는 하나의 명령어를 여러 데이터에 동시에 적용하는 병렬 처리 방식이다. GPU의 벡터·행렬 연산, 멀티미디어 처리 등 동일 연산을 대량 데이터에 반복 적용할 때 효율적이다.

따라서 "하나의 명령어를 동시에 여러 데이터에 적용, 벡터 연산 활용"인 ④번이 옳다. ①(서로 다른 명령어=MIMD), ②(다중 명령 동시 실행=슈퍼스칼라), ③(단일 명령이 여러 독립 연산 포함=VLIW 유사)은 SIMD의 정의가 아니다.

오답 분석

보기①

각 코어가 서로 다른 명령어를 실행하는 것은 MIMD에 해당한다.

보기②

한 클록에 여러 명령어를 동시 실행하는 것은 슈퍼스칼라 구조 설명이다.

보기③

하나의 명령어가 여러 독립 연산을 포함하는 것은 VLIW 계열 설명이다.

보기④

정답. 단일 명령을 다중 데이터에 적용(벡터 연산) = SIMD.

관련 이론 정리

플린(Flynn) 분류는 명령어 스트림과 데이터 스트림의 다중성에 따라 컴퓨터 구조를 네 가지로 나눈다. SISD는 단일 명령·단일 데이터로 전통적 순차 처리기이고, SIMD는 하나의 명령을 다수 데이터에 동시에 적용해 벡터·행렬·이미지 같은 데이터 병렬 작업에 강하다(GPU, 벡터 프로세서). MISD는 여러 명령이 한 데이터에 작용하는 드문 형태(일부 파이프라인·내고장 시스템)이며, MIMD는 여러 명령이 여러 데이터에 독립적으로 작용하는 멀티코어·클러스터 구조로 태스크 병렬에 적합하다. 이와 구분해야 할 개념으로, 한 클록에 여러 명령을 동시 발행하는 슈퍼스칼라, 한 명령어에 여러 연산을 묶는 VLIW가 있다. 감리에서는 워크로드가 데이터 병렬(SIMD/GPU)인지 태스크 병렬(MIMD)인지에 따라 아키텍처 선택이 타당한지 검토한다.

MIMD 다중 명령·다중 데이터(멀티코어)
실무 적용 사례

병렬 처리·AI 가속 시스템 감리 시 워크로드 특성(데이터 병렬 vs 태스크 병렬)에 맞는 아키텍처(SIMD/GPU vs MIMD/멀티코어) 선택이 적절한지 검토한다.

감리사 시험 출제 포인트

플린 분류에서 SIMD(단일 명령·다중 데이터)를 MIMD·슈퍼스칼라·VLIW와 구분한다. "벡터 연산"이 SIMD 단서다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

평균 메모리 접근 시간 = (캐시 접근 시간 + 실패율 × 실패 손실)이며, 이를 클록 단위로 계산한 뒤 클록 사이클 시간을 곱해 ns로 환산한다.

  • 캐시 접근 시간 = 2 클록
  • 실패율 × 실패 손실 = 0.05 × 40 = 2 클록
  • 평균 접근 시간(클록) = 2 + 2 = 4 클록
  • ns 환산 = 4 클록 × 2 ns/클록 = 8 ns

따라서 정답은 ④번 8 ns이다.

오답 분석

보기①

2 ns는 캐시 접근 시간(2클록)을 잘못 환산하거나 미스 페널티를 빠뜨린 값이다.

보기②

4 ns는 캐시 접근(2클록)만 ns 환산한 값(미스 페널티 미반영)이다.

보기③

6 ns는 클록·ns 환산이나 미스율 계산을 잘못한 값이다.

보기④

정답. (2 + 0.05×40) × 2 = 4 × 2 = 8 ns.

관련 이론 정리

메모리 계층(레지스터-캐시-주기억-보조기억)은 빠르지만 비싼 상위 계층에 자주 쓰는 데이터를 두어 평균 접근 시간을 낮춘다. 캐시의 성능은 평균 메모리 접근 시간(AMAT) = 히트 시간 + 미스율 × 미스 페널티로 표현된다. 히트 시간은 캐시에서 바로 읽는 시간, 미스 페널티는 캐시에 없어 하위 계층에서 가져오는 데 드는 추가 시간이다. 예컨대 캐시 접근 2클록, 미스율 5%, 미스 페널티 40클록이면 AMAT = 2 + 0.05×40 = 4클록이고, 클록 사이클이 2ns면 8ns가 된다. 미스율을 낮추려면 캐시 크기·연관도·블록 크기를 조정하고(3C: Compulsory·Capacity·Conflict 미스), 미스 페널티는 다단계 캐시·프리페치로 완화한다. 감리에서는 캐시 계층 설계가 성능 목표(응답시간)를 충족하는지 정량 근거로 확인한다.

평균 메모리 접근 시간:

요소 값
캐시 접근 2 클록
미스율 × 페널티 0.05 × 40 = 2 클록
합(클록) 4 클록
ns 환산 4 × 2 = 8 ns
실무 적용 사례

성능 튜닝 감리 시 캐시 히트율·미스 페널티로 평균 메모리 접근 시간을 산정해 캐시 계층·크기 설계가 성능 목표에 부합하는지 검토한다.

감리사 시험 출제 포인트

평균 접근 시간 = 히트 시간 + 미스율 × 미스 페널티 공식을 클록·ns 환산과 함께 정확히 적용한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설
  • LFU(Least Frequently Used): 사용 빈도(횟수)가 가장 적은 페이지를 교체한다.
  • LRU(Least Recently Used): 가장 오랫동안 사용되지 않은(최근 참조 시점이 가장 오래된) 페이지를 교체한다.

④는 LFU를 "최근까지 가장 오랫동안 사용하지 않은 페이지를 교체"라고 설명했는데, 이는 LRU의 정의이다. LFU는 사용 빈도 기준이므로 ④가 옳지 않다. 정답은 ④번이다.

①(가상→물리 주소 매핑), ②(물리주소), ③(페이지 부재)는 가상 기억장치의 옳은 설명이다.

보기①

보기②

옳음. 실제 적재되는 주기억장치 주소가 물리주소다.

보기③

옳음. 참조 페이지가 주기억장치에 없으면 페이지 부재가 발생한다.

보기④

정답(부적절). "가장 오랫동안 사용 안 함"은 LRU이며, LFU는 사용 빈도 최소 페이지를 교체한다.

관련 이론 정리

가상 기억장치는 프로그램이 사용하는 논리적 가상주소 공간을 물리 메모리보다 크게 제공하고, 필요한 페이지만 주기억장치에 적재(요구 페이징)한다. 참조한 페이지가 메모리에 없으면 페이지 부재(page fault)가 발생해 보조기억장치에서 페이지를 가져오는데, 이때 빈 프레임이 없으면 어떤 페이지를 내보낼지 정하는 것이 페이지 교체 알고리즘이다. FIFO는 가장 먼저 적재된 페이지를, LRU는 가장 오랫동안 참조되지 않은(최근 사용 시점이 가장 오래된) 페이지를, LFU는 참조 빈도(횟수)가 가장 적은 페이지를 교체한다. 이론상 최적인 OPT는 미래 참조를 알아야 해 구현 불가하며 성능 비교 기준으로 쓰인다. 교체 정책이 부적절하거나 프레임이 부족하면 페이지 부재가 폭증하는 스래싱이 일어나므로, 감리에서는 작업 집합(working set) 기반 프레임 할당과 교체 정책을 점검한다.

감리사 시험 출제 포인트

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

데이터 통신의 오류 검출 방식으로는 패리티 비트(④), 블록 검사(BSC/블록합, ②), 다항 코드(CRC, ③) 등이 사용된다.

① BCD 코드(Binary Coded Decimal)는 10진수 한 자리를 4비트 2진으로 표현하는 데이터 표현 방식일 뿐 오류 검출 목적의 코드가 아니다. 따라서 오류 검출 방식으로 가장 적합하지 않은 것은 ①번이다.

오답 분석

보기①

정답(부적합). BCD는 10진수 표현 코드로 오류 검출 방식이 아니다.

보기②

블록 검사(블록합 검사)는 여러 문자의 패리티를 종합하는 오류 검출 방식이다.

보기③

다항 코드(CRC)는 생성 다항식을 이용한 강력한 오류 검출 방식이다.

보기④

패리티 비트는 1비트 오류를 검출하는 기본 방식이다.

관련 이론 정리

데이터 통신에서 전송 중 발생하는 비트 오류를 찾아내는 방법이 오류 검출이며, 검출만 하는 방식과 검출+정정을 하는 방식으로 나뉜다. 패리티 비트는 1의 개수를 짝수/홀수로 맞춰 1비트 오류를 검출하는 가장 단순한 방식이고, 블록 검사(BSC, 블록합/2차원 패리티)는 여러 문자에 대해 가로·세로 패리티를 종합해 검출력을 높인다. CRC(순환 중복 검사, 다항 코드)는 생성 다항식으로 나눈 나머지를 검사값으로 붙여 버스트 오류까지 강력하게 검출하며 데이터링크 계층에서 널리 쓰인다. 해밍 코드는 검출을 넘어 오류 위치를 찾아 정정까지 한다. 반면 BCD 코드는 10진수를 4비트로 표현하는 데이터 표현 코드일 뿐 오류 검출 기법이 아니다. 감리에서는 매체·거리·신뢰도 요구에 맞는 검출 방식과 재전송(ARQ)/전진오류정정(FEC) 정책을 확인한다.

오류 검출 방식:

방식 특징
패리티 1비트 오류 검출
블록 검사 문자 블록 종합 검사
CRC(다항코드) 강력한 검출(버스트 오류)
해밍코드 검출+정정
실무 적용 사례

통신 신뢰성 감리 시 링크·계층별 오류 검출(CRC 등) 방식이 적용됐는지, 검출 능력이 요구 신뢰도를 충족하는지 확인한다.

감리사 시험 출제 포인트

오류 검출 방식(패리티·블록검사·CRC)과 데이터 표현 코드(BCD)를 구분한다. BCD를 오류 검출 방식으로 오인하지 않도록 한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

데이터 압축은 데이터 내 중복(redundancy)을 제거해 크기를 줄인다. 따라서 중복 정보가 많을수록 압축률이 높아진다(더 많이 줄일 수 있음).

②는 "중복 정보가 많으면 압축률이 낮다"고 서술했으나, 실제로는 중복이 많으면 압축률이 높다. 방향이 반대이므로 ②가 적절하지 않다. 정답은 ②번이다.

①(압축 후 전송이 유리)·③(비손실=내용 보존)·④(손실=원본과 다름)는 옳은 설명이다.

오답 분석

보기①

옳음. 압축으로 크기를 줄이면 전송량이 감소해 전송 속도 면에서 유리하다.

보기②

정답(부적절). 중복이 많으면 압축률이 높다(낮다는 반대).

보기③

옳음. 비손실(무손실) 압축은 원래 데이터를 완전히 복원한다.

보기④

옳음. 손실 압축은 복원 시 원본과 완전히 동일하지 않다.

관련 이론 정리

데이터 압축은 데이터에 내재된 중복(redundancy)을 제거하거나 사람이 인지하기 어려운 정보를 버려 크기를 줄이는 기술로, 저장·전송 효율을 높인다. 따라서 데이터에 반복·규칙적 패턴 같은 중복이 많을수록 압축률이 높아진다(더 많이 줄어든다). 크게 비손실(무손실) 압축은 원본을 완전히 복원할 수 있어 문서·실행파일에 쓰이며(ZIP, PNG, GZIP, 허프만·LZ 계열), 손실 압축은 일부 정보를 버려 높은 압축률을 얻지만 복원 시 원본과 완전히 같지 않아 이미지·오디오·영상에 쓰인다(JPEG, MP3, H.264). 압축은 CPU 연산과 전송량의 트레이드오프이므로, 대용량·저대역 환경에서는 압축 후 전송이 유리하다. 감리에서는 데이터 특성(정확도 요구·중복성)에 맞는 압축 방식과 압축률·성능이 적절한지 검토한다.

손실 근사 복원(JPEG, MP3)
목적 저장·전송 효율

데이터 전송·저장 최적화 감리 시 데이터 특성(중복성·정확도 요구)에 맞는 압축(비손실/손실) 방식과 압축률이 적절한지 검토한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

파장 λ = 신호 속도(v) / 주파수(f)이다.

λ = (3×10⁸ m/s) / (3×10³ Hz) = 1×10⁵ m = 100,000 m

주파수 3KHz = 3×10³ Hz를 사용하면 파장은 10⁵ m = 100,000m이다. 따라서 정답은 ③번이다.

오답 분석

보기①

1,000m는 주파수를 3MHz(10⁶)로 잘못 대입한 값이다.

보기②

10,000m는 주파수를 30KHz로 잘못 대입한 값이다.

보기③

정답. 3×10⁸ / 3×10³ = 10⁵ = 100,000m.

보기④

1,000,000m는 주파수를 300Hz로 잘못 대입한 값이다.

관련 이론 정리

파동(신호)은 속도(v), 주파수(f), 파장(λ)의 관계 v = f × λ로 기술되며, 여기서 파장은 신호 한 주기가 공간에서 차지하는 길이다. 따라서 파장은 λ = v / f로 구하고, 속도가 일정하면 주파수가 높을수록 파장은 짧아진다(반비례). 전자기파의 자유공간 속도는 약 3×10⁸ m/s이며, 계산 시 단위 환산(KHz=10³, MHz=10⁶, GHz=10⁹ Hz)에 유의해야 한다. 예로 3KHz 신호는 λ=3×10⁸/3×10³=10⁵ m=100,000 m이다. 파장은 안테나 크기(대략 파장의 1/2 또는 1/4), 회절·투과 특성과 직결되어 무선·광 통신 설계의 기초가 된다. 감리에서는 사용 주파수 대역과 전파 특성이 서비스 요구(거리·장애물)에 부합하는지 검토한다.

파장 공식:

항목 값
파장 λ v / f
속도 v 3×10⁸ m/s
주파수 f 3×10³ Hz(3KHz)
결과 10⁵ m = 100,000m
실무 적용 사례

무선·광 통신 설계 감리 시 주파수-파장 관계로 안테나 크기·전파 특성을 이해해 통신 규격 적정성을 검토한다.

감리사 시험 출제 포인트

λ = v/f 공식과 단위 환산(KHz=10³)을 정확히 적용한다. 주파수 자릿수 오류가 함정이다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

지터(jitter)는 패킷마다 지연 시간이 달라 패킷 도착 시간의 편차(변동)가 발생하는 현상이다. 문제의 "서로 다른 지연시간, 고르지 못한 지연, 도착 시간 편차, 간격 불규칙"은 정확히 지터의 정의이므로 정답은 ①번이다.

지연시간(③ latency)은 지연의 절대 크기, 전파 지연(④)은 매체 전파에 걸리는 시간, 패킷 손실(② packet loss)은 패킷이 버려지는 현상으로 각각 다른 개념이다.

오답 분석

보기①

정답. 지연 시간의 변동(편차) = 지터.

보기②

패킷 손실은 패킷이 목적지에 도달하지 못하는 현상이다.

보기③

지연시간(latency)은 지연의 크기 자체로, 변동(편차)이 아니다.

보기④

전파 지연은 신호가 매체를 통과하는 데 걸리는 시간이다.

관련 이론 정리

네트워크 전송 품질(QoS)은 여러 지연·손실 지표로 측정된다. 지연(latency)은 패킷이 출발지에서 목적지까지 도달하는 데 걸리는 총 시간으로, 전파 지연(매체 통과)·전송 지연(대역폭 대비 크기)·처리·큐잉 지연의 합이다. 지터(jitter)는 이 지연이 패킷마다 달라 도착 간격이 불규칙해지는 지연의 변동(편차)을 뜻한다. 패킷 손실(packet loss)은 혼잡·오류로 패킷이 폐기되는 현상이다. 특히 VoIP·실시간 영상처럼 일정한 도착 간격이 중요한 서비스는 지터에 민감해, 수신측이 패킷을 잠시 모아 균일하게 재생하는 지터 버퍼로 완화하고, 우선순위 큐잉·대역폭 보장 같은 QoS로 지연·손실을 관리한다. 감리에서는 SLA의 지연·지터·손실 목표와 QoS 설계의 정합성을 확인한다.

지터(jitter) 지연의 변동(편차)
전파 지연 매체 전파 시간
패킷 손실 패킷 폐기
실무 적용 사례

실시간 서비스(VoIP·영상) 품질 감리 시 지터 버퍼·QoS 설정으로 지연 변동을 완화해 통화·재생 품질을 보장하는지 검토한다.

감리사 시험 출제 포인트

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

POP3(Post Office Protocol 3)는 메일 수신(다운로드) 프로토콜로, 명령에는 USER, PASS, STAT, LIST(메시지 목록/크기 조회), RETR, DELE, QUIT 등이 있다. LIST + <일련번호>는 특정 메시지의 정보를 조회하는 POP3 명령이므로 ②번이 옳다.

①HELO, ③MAIL, ④RCPT는 메일 발신 프로토콜인 SMTP의 명령이다. 따라서 POP3 명령은 ②번이다.

오답 분석

보기①

HELO는 SMTP에서 세션을 시작하며 송신 호스트를 식별하는 명령이다.

보기②

정답. LIST는 POP3에서 메시지 목록/정보를 조회하는 명령이다.

보기③

MAIL은 SMTP에서 송신자를 지정하는 명령(MAIL FROM)이다.

보기④

RCPT는 SMTP에서 수신자를 지정하는 명령(RCPT TO)이다.

관련 이론 정리

전자우편은 발신과 수신에 서로 다른 프로토콜을 사용한다. SMTP(발신)는 메일 서버 간 또는 클라이언트→서버로 메일을 보내는 프로토콜로, HELO/EHLO(세션 시작), MAIL FROM(송신자), RCPT TO(수신자), DATA(본문), QUIT 명령을 사용한다. POP3(수신)는 서버에 도착한 메일을 클라이언트로 내려받는 프로토콜로, USER/PASS(인증), STAT(총계), LIST(메시지 목록·크기), RETR(가져오기), DELE(삭제), QUIT 명령을 쓰며 보통 다운로드 후 서버에서 삭제한다. IMAP(수신)은 메일을 서버에 두고 여러 기기에서 동기화·검색·폴더 관리를 지원하는 점이 POP3와 다르다. 따라서 HELO·MAIL·RCPT는 SMTP, LIST·RETR·DELE는 POP3 명령이다. 감리에서는 발신(SMTP)·수신(POP3/IMAP) 구성, 포트, 인증·암호화(SMTPS/STARTTLS) 적용을 확인한다.

메일 프로토콜 명령:

프로토콜 명령
POP3(수신) USER, PASS, STAT, LIST, RETR, DELE, QUIT
SMTP(발신) HELO, MAIL, RCPT, DATA, QUIT
IMAP(수신) LOGIN, SELECT, FETCH 등
실무 적용 사례

메일 시스템 감리 시 발신(SMTP)·수신(POP3/IMAP) 프로토콜과 포트·인증·명령이 규격에 맞게 구성됐는지 확인한다.

감리사 시험 출제 포인트

POP3(수신) 명령(LIST 등)과 SMTP(발신) 명령(HELO/MAIL/RCPT)을 구분한다. 발신 명령을 POP3로 오인하지 않도록 한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

CRC 하드웨어에서 시프트 레지스터의 단수는 생성 다항식의 최고 차수와 같다. 그림은 4단 레지스터이므로 최고 차수는 x⁴이다.

피드백(XOR) 게이트의 위치가 다항식의 각 항을 결정한다. 데이터 입력단의 XOR는 최고차항(x⁴)과 상수항(1)에 대응하고, 레지스터 중간(2번째와 3번째 단 사이)의 XOR는 x² 항에 대응한다. 따라서 생성 다항식은 x⁴ + x² + 1이며 정답은 ①번이다.

오답 분석

보기①

정답. 4단 레지스터(x⁴) + 중간 XOR(x²) + 상수항(1) = x⁴ + x² + 1.

보기②

x² + 1은 2단 레지스터에 해당해 4단 구성과 맞지 않는다.

보기③

x⁴ + x³ + x는 XOR 위치(x³, x¹)가 그림과 다르다.

보기④

x³ + x는 최고차수가 3으로 4단 레지스터와 맞지 않는다.

관련 이론 정리

CRC는 데이터를 미리 약속한 생성 다항식 G(x)로 나눈 나머지(FCS)를 검사값으로 붙이는 오류 검출 기법으로, 하드웨어로는 시프트 레지스터와 XOR(배타적 OR) 게이트로 나눗셈(모듈로-2 연산)을 구현한다. 이때 레지스터의 단(段) 수가 생성 다항식의 최고 차수와 같고, 각 XOR 게이트의 위치가 다항식의 항에 대응한다. 예를 들어 4단 레지스터는 x⁴을 뜻하고, 데이터 입력단의 XOR는 최고차항(x⁴)과 상수항(1)을, 2·3단 사이의 XOR는 x² 항을 나타내어 G(x)=x⁴+x²+1이 된다. 실무 표준으로 CRC-16(x¹⁶+x¹⁵+x²+1), CRC-32(이더넷) 등이 있으며 버스트 오류 검출에 강하다. 감리에서는 링크·저장 계층의 CRC 규격이 요구 신뢰도(검출 능력)에 부합하는지 확인한다.

실무 적용 사례

통신·저장 오류 검출 감리 시 CRC 생성 다항식(CRC-16, CRC-32 등)과 하드웨어 구현이 규격에 부합해 버스트 오류를 충분히 검출하는지 검토한다.

관련 이론 정리

IEEE 802 위원회는 LAN/MAN의 물리·데이터링크 계층 표준을 번호로 관리한다. 802.1 계열은 브리징·관리로, 그중 802.1Q는 이더넷 프레임에 4바이트 VLAN 태그(TPID+VLAN ID 등)를 삽입해 여러 스위치에 걸쳐 논리적 VLAN을 구분·전달하는 프레임 태깅 표준이고, 802.1D는 루프를 막는 STP(스패닝 트리)이다. 802.3은 이더넷(유선 LAN)으로 그중 802.3af/at는 랜 케이블로 전력을 공급하는 PoE다. 802.11은 무선랜(Wi-Fi), 802.15.1은 블루투스(WPAN), 802.16은 WiMAX다. VLAN은 브로드캐스트 도메인을 분리해 성능·보안·관리성을 높이며, 스위치 간 태그를 실어 나르는 링크를 트렁크라 한다. 감리에서는 VLAN 설계(802.1Q 트렁크·네이티브 VLAN)와 세그먼트 격리가 요구에 맞는지 확인한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

SDN은 제어 평면(control plane)과 데이터 평면(data plane)을 분리하고, 중앙 컨트롤러가 데이터 평면(스위치)의 포워딩을 프로그래밍한다. 컨트롤러와 데이터 평면(스위치)을 잇는 Southbound Interface의 대표 프로토콜이 OpenFlow이다.

③은 "Control-Data Plane Interface(= Southbound)가 데이터 평면의 포워딩 제어·정보 수집용 인터페이스이며 OpenFlow가 대표 프로토콜"이라 정확히 서술했으므로 옳다. 정답은 ③번이다.

①(제어-응용 평면은 Northbound로 연결, Southbound 아님), ②(컨트롤러는 장비 내장이 아니라 중앙 분리), ④(데이터 평면은 컨트롤러 제어를 받으며 독립 동작 아님)는 모두 틀린 설명이다.

관련 이론 정리

SDN(소프트웨어 정의 네트워킹)은 전통적으로 네트워크 장비에 함께 있던 제어 평면(경로 결정)과 데이터 평면(패킷 포워딩)을 분리하고, 논리적으로 중앙 집중된 컨트롤러가 다수의 스위치를 프로그래밍하는 구조다. 구조는 위에서부터 응용 평면(네트워크 앱)-제어 평면(컨트롤러)-데이터 평면(스위치)으로 나뉘고, 이들을 잇는 인터페이스가 있다. Northbound 인터페이스는 제어 평면과 응용 평면을 잇는 API로 애플리케이션이 네트워크 정책을 요청하고, Southbound 인터페이스는 컨트롤러와 데이터 평면(스위치)을 잇는데 OpenFlow가 대표 프로토콜로 포워딩 규칙 설정·정보 수집을 담당한다. 이를 통해 정책 기반 자동화·중앙 가시성·프로그래머빌리티를 얻지만 컨트롤러가 단일 장애점이 될 수 있어 이중화가 중요하다. 감리에서는 평면 분리·컨트롤러 가용성·정책 일관성을 점검한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

netstat -an은 모든(-a) 활성 연결과 리스닝 포트를 이름 해석 없이 숫자(-n)로 표시한다. 백도어/트로이목마가 열어둔 비정상적인 연결이나 리스닝 포트를 빠르게 확인할 수 있어 침해 여부 점검에 적합하다. 따라서 정답은 ①번이다.

-r(②③ 중 -r)는 라우팅 테이블, -s는 프로토콜별 통계, -p는 (환경에 따라) 특정 프로토콜/프로세스 표시로 연결·포트 전수 확인 목적에는 -an이 가장 적절하다.

오답 분석

보기①

정답. -an으로 모든 연결·리스닝 포트를 숫자로 확인 → 백도어 포트 탐지.

보기②

-p는 특정 프로토콜/프로세스 관련 옵션으로 전체 연결 전수 확인에는 -an이 낫다.

보기③

-r은 라우팅 테이블을 표시하는 옵션으로 연결 확인 용도가 아니다.

보기④

-s는 프로토콜별 통계를 표시하는 옵션이다.

관련 이론 정리

netstat(network statistics)은 호스트의 네트워크 연결·라우팅·인터페이스·프로토콜 통계를 보여주는 진단 명령으로, 침해사고 대응에서 백도어의 비인가 연결·리스닝 포트를 찾는 데 유용하다. -a는 모든 활성 연결과 리스닝(대기) 포트를, -n은 호스트명·서비스명을 해석하지 않고 숫자(IP·포트)로 빠르게 표시하므로, 둘을 합친 -an이 전체 연결·개방 포트를 한눈에 점검하는 데 적합하다. -r은 라우팅 테이블, -s는 프로토콜별(TCP/UDP/ICMP 등) 통계, -p(플랫폼별)는 특정 프로토콜이나 연결을 소유한 프로세스를 보여준다. 실무에서는 netstat(또는 ss)로 의심 포트를 찾고, 이를 프로세스·서비스와 매핑해 악성코드를 식별한다. 감리에서는 서버의 개방 포트·서비스 최소화와 이상 연결 모니터링 절차를 확인한다.

netstat 주요 옵션:

옵션 내용
-a 모든 연결·리스닝 포트
-n 숫자 표시(이름 해석 안 함)
-r 라우팅 테이블
-s 프로토콜별 통계
실무 적용 사례

침해사고 대응 감리 시 netstat -an으로 비인가 연결·리스닝 포트를 확인하고, 프로세스 매핑으로 악성 프로그램을 식별하는 절차가 수립됐는지 검토한다.

감리사 시험 출제 포인트

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

경사소실(gradient vanishing)은 심층 신경망에서 역전파 시 기울기가 하위 층으로 갈수록 점점 작아져 학습이 정체되는 문제다. 은닉층을 증가시키면 오히려 경사소실이 더 심해진다. 해결책은 ReLU 활성함수, 배치 정규화, 잔차 연결(ResNet), 적절한 초기화 등이다.

④는 "은닉층을 증가시켜 경사소실을 해결"한다고 했으나 이는 반대로 문제를 악화시키므로 적절하지 않다. 정답은 ④번이다. ①(과적합), ②(정규화 규제), ③(역전파)는 옳은 설명이다.

오답 분석

보기①

옳음. 과적합은 학습 데이터를 과하게 학습해 일반화가 떨어지는 문제다.

보기②

옳음. L1/L2 정규화(가중치 규제)는 과적합 완화 방법이다.

보기③

옳음. 역전파는 손실함수의 기울기를 계산해 가중치를 갱신한다.

보기④

정답(부적절). 은닉층 증가는 경사소실을 악화시킨다. 해결책은 ReLU·배치정규화·잔차연결 등이다.

관련 이론 정리

기계학습 모델 학습에서는 여러 문제가 성능을 저해한다. 과적합(overfitting)은 모델이 학습 데이터의 잡음·세부까지 외워 학습 성능은 좋지만 새로운 데이터 일반화가 떨어지는 현상으로, L1/L2 정규화(가중치 규제)·드롭아웃·조기 종료·데이터 증강으로 완화한다. 심층 신경망은 역전파(backpropagation)로 손실함수의 기울기를 출력층에서 입력층 방향으로 전파하며 가중치를 갱신하는데, 층이 깊으면 기울기가 점점 작아져 하위 층이 거의 학습되지 않는 경사소실(gradient vanishing)이 생긴다. 이는 은닉층을 더 늘리면 오히려 악화되며, 해결책은 ReLU 계열 활성함수, 배치 정규화, 잔차 연결(ResNet), 적절한 가중치 초기화다. 반대로 기울기가 폭증하는 경사폭발은 그래디언트 클리핑으로 억제한다. 감리에서는 검증·테스트 성능으로 일반화를 확인하고 과적합·경사 문제 대응이 적용됐는지 본다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

비지도 학습(unsupervised learning)은 정답(레이블)이 없는 데이터에서 패턴·구조를 찾는 방식으로, 대표적으로 군집화(clustering)가 있다. K-평균 군집화는 데이터를 k개 군집으로 묶는 비지도 학습 알고리즘이다.

②나이브 베이즈 분류·③서포트 벡터 머신은 분류(지도 학습), ④선형 회귀는 회귀(지도 학습)이다. 따라서 비지도 학습은 ①번 K-평균 군집화이다.

오답 분석

보기①

보기②

나이브 베이즈 분류는 레이블을 사용하는 지도 학습(분류)이다.

보기③

서포트 벡터 머신은 지도 학습(분류/회귀)이다.

보기④

선형 회귀는 지도 학습(회귀)이다.

관련 이론 정리

기계학습은 학습에 사용하는 정답(레이블)의 유무와 피드백 방식에 따라 유형이 나뉜다. 지도 학습(supervised)은 입력-정답 쌍으로 학습해 새 입력의 결과를 예측하며, 출력이 범주면 분류(나이브 베이즈, SVM, 결정트리, 로지스틱 회귀), 연속값이면 회귀(선형 회귀)이다. 비지도 학습(unsupervised)은 정답 없이 데이터의 숨은 구조를 찾으며 대표적으로 군집화(K-평균, 계층적 군집, DBSCAN)와 차원 축소(PCA)가 있다. 강화 학습(reinforcement)은 에이전트가 환경과 상호작용하며 보상을 최대화하도록 정책을 학습한다(Q-러닝 등). 따라서 K-평균은 레이블이 없는 비지도 군집화이고, SVM·나이브 베이즈·선형 회귀는 지도 학습이다. 감리에서는 문제 성격(레이블 유무·목표)에 맞는 학습 방식과 평가 지표 선택이 타당한지 확인한다.

비지도 K-평균, 계층적 군집, PCA
강화 Q-러닝 등
실무 적용 사례

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

큐비트는 결어긋남(decoherence)·잡음에 매우 민감해 전송·연산 과정에서 오류가 빈번히 발생하며, 이를 보정하기 위해 양자 오류 정정(QEC)이 활발히 연구된다.

④는 "큐비트 전송 과정에서 오류가 발생하지 않는 장점"이라 했으나, 실제로는 오류가 발생하기 쉬워 오류 정정이 큰 과제이므로 적절하지 않다. 정답은 ④번이다.

①(쇼어 알고리즘=소인수분해→위수/차수 찾기), ②(양자 병렬성=중첩으로 동시 처리), ③(측정 시 상태 붕괴=값에 영향)은 옳은 설명이다.

오답 분석

보기①

옳음. 쇼어 알고리즘은 소인수분해를 위수(차수) 찾기 문제로 환원한다.

보기②

옳음. 중첩을 이용해 여러 입력을 동시에 평가하는 양자 병렬성을 갖는다.

보기③

옳음. 중첩 상태를 측정하면 상태가 붕괴되어 값에 영향을 준다.

보기④

정답(부적절). 큐비트는 오류에 취약해 양자 오류 정정이 필요하다. "오류 없음"은 틀림.

관련 이론 정리

양자 컴퓨팅은 0과 1을 동시에 확률적으로 표현하는 큐비트(qubit)를 기본 단위로 한다. 큐비트는 여러 상태를 동시에 갖는 중첩(superposition) 덕분에 n개 큐비트로 2ⁿ 상태를 병렬 표현·연산할 수 있고, 큐비트 간 얽힘(entanglement)으로 강한 상관을 만든다. 다만 측정하는 순간 중첩이 하나의 값으로 붕괴되므로 결과를 얻는 방식이 고전 컴퓨터와 다르다. 큐비트는 외부 잡음·결어긋남(decoherence)에 매우 취약해 오류가 빈번하며, 이를 보정하는 양자 오류 정정(QEC)과 물리 큐비트를 묶어 논리 큐비트를 만드는 연구가 핵심 과제다. 대표 알고리즘으로 소인수분해를 위수(차수) 찾기로 환원해 RSA를 위협하는 쇼어 알고리즘, 비정렬 탐색을 가속하는 그로버 알고리즘이 있다. 감리에서는 양자 위협 대비 양자내성암호(PQC) 전환 로드맵을 검토한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

파인튜닝(fine-tuning)은 대규모로 사전 훈련(pre-training)된 LLM을, 특정 작업/도메인에 맞는 상대적으로 작은 데이터셋으로 추가 학습해 파라미터를 조정하는 기법이다. 사전 훈련된 언어 지식을 활용하면서 특정 작업 성능을 크게 높인다.

문제의 설명은 정확히 파인튜닝이므로 정답은 ④번이다. ①어텐션(입력 요소 간 중요도 가중), ②임베딩(토큰의 벡터 표현), ③토큰화(텍스트를 토큰으로 분할)는 각각 다른 개념이다.

오답 분석

보기①

어텐션은 입력 시퀀스 내 요소 간 관련성(가중치)을 계산하는 메커니즘이다.

보기②

임베딩은 토큰을 밀집 벡터로 표현하는 것이다.

보기③

토큰화는 텍스트를 토큰(단어/서브워드)으로 분할하는 전처리다.

보기④

정답. 사전학습 모델을 특정 작업 데이터로 추가 조정 = 파인튜닝.

관련 이론 정리

대규모 언어모델(LLM)은 방대한 일반 텍스트로 사전학습(pre-training)해 언어의 통계적 지식을 습득한 뒤, 목적에 맞게 활용된다. 파인튜닝(fine-tuning)은 이 사전학습 모델을 특정 작업·도메인의 상대적으로 작은 레이블 데이터로 추가 학습해 파라미터를 조정, 해당 작업 성능을 끌어올리는 방법이다(전체 미세조정, LoRA 같은 경량 방식 등). 이와 구분해야 할 핵심 구성요소로, 텍스트를 토큰(단어·서브워드) 단위로 나누는 토큰화, 토큰을 의미 벡터로 바꾸는 임베딩, 시퀀스 내 요소 간 관련도를 가중치로 계산해 문맥을 반영하는 어텐션(트랜스포머의 핵심)이 있다. 최근에는 재학습 없이 외부 지식을 검색해 붙이는 RAG, 지시문만 주는 프롬프트 방식도 널리 쓰인다. 감리에서는 파인튜닝·RAG 등 방식 선택과 학습 데이터 품질·편향·비용·평가가 적절한지 검토한다.
