영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

ISO 8000의 데이터 품질 기준은 크게 유효성(validity)과 활용성(usability)으로 구분된다.
- 유효성: 데이터 값 자체의 정확성·정합성 측면 → 사실성(정확성), 정합성(적합성), 필수성(완전성) 등.
- 활용성: 데이터를 실제로 활용하는 관점 → 충분성, 유용성, 접근성, 적시성, 보안성 등.

따라서 활용성에 해당하는 것은 충분성으로 정답은 ④번이다. 사실성·정합성·필수성은 유효성 범주에 속한다.

오답 분석

보기①

사실성(정확성)은 데이터 값이 실제와 일치하는지에 대한 유효성 기준이다.

보기②

필수성은 필수 데이터의 존재 여부(완전성)로 유효성 범주다.

보기③

정합성은 표준·규칙 부합 여부로 유효성 범주다.

보기④

정답. 충분성은 데이터를 활용하기에 충분한지에 대한 활용성 기준이다.

관련 이론 정리

ISO 8000 품질 기준 분류:

구분 세부 기준
유효성 사실성(정확성), 정합성, 필수성 등
활용성 충분성, 유용성, 접근성, 적시성, 보안성
실무 적용 사례

데이터 품질 진단 감리 시 값의 정확성(유효성)뿐 아니라 활용 관점(충분성·적시성·접근성)까지 평가해 데이터가 실제 의사결정에 쓰일 수 있는지 점검한다.

감리사 시험 출제 포인트

ISO 8000의 유효성/활용성 두 축과 각 세부 기준의 소속을 구분한다. "충분성=활용성"을 기억한다.

1:1 관계의 속성(StartDate)은 다음 세 가지로 배치할 수 있다. (1) 관계 타입에 그대로 유지(③), (2) 한쪽 엔티티(EMPLOYEE)로 이동(①), (3) 다른 쪽 엔티티(DEPARTMENT)로 이동(②). 1:1이므로 어느 한쪽에 두어도 각 인스턴스가 유일하게 대응되어 중복이 없다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

후보키는 전체 속성(A~G)을 결정하는 최소 속성 집합이다. 각 보기의 폐포를 계산한다.

  • ① A: A → ABCDEFG로 전체 결정 → 후보키. ✓
  • ② BD: BD → E로 {B,D,E}까지. CE → A는 C가 필요한데 없어 A를 유도 못 함. BD+ = {B,D,E}로 전체를 결정하지 못한다 → 슈퍼키조차 아니므로 후보키가 아니다. ✗
  • ③ CE: CE → A, A → 전체 → CE+ = 전체. 최소성 만족(C·E 단독 불가) → 후보키. ✓
  • ④ BCD: BD → E로 E, CE → A로 A(C·E 보유), A → 전체 → BCD+ = 전체. 부분집합(BD, CD, BC) 어느 것도 전체를 결정 못 해 최소 → 후보키. ✓

따라서 후보키가 아닌 것은 ②번 BD이다.

오답 분석

보기①

A는 단독으로 전체를 결정(A→ABCDEFG)하는 후보키다.

보기②

정답(후보키 아님). BD+ = {B,D,E}로 A·C·F·G를 결정하지 못한다(C가 없어 CE→A 적용 불가).

보기③

CE는 CE→A→전체로 후보키다.

보기④

BCD는 BD→E, CE→A, A→전체로 전체를 결정하고 최소성도 만족하는 후보키다.

관련 이론 정리

후보키 폐포 판정:

후보키 조건 내용
유일성 폐포가 전체 속성
최소성 진부분집합은 전체 결정 불가
슈퍼키 유일성만(최소성 불필요)
BD 폐포 {B,D,E}로 부족 → 키 아님
실무 적용 사례

키 설계 감리 시 폐포 계산으로 후보키를 검증하고, 전체를 결정하지 못하는 속성 조합을 키로 잘못 지정하지 않았는지 확인한다.

감리사 시험 출제 포인트

각 후보 집합의 폐포를 계산해 전체 속성 결정 여부를 판정한다. BD처럼 폐포가 부족한(슈퍼키도 아닌) 집합을 골라낸다.

문제

SQL에서 사용되는 COUNT 함수에 대한 다음 설명 중 옳은 것으로 묶인 것은?

가. 중복되는 값이 있는 속성의 경우, COUNT(속성명)의 결과와 COUNT(DISTINCT 속성명)의 결과는 동일하다.
나. 중복되는 값이 없는 속성의 경우, COUNT(*)의 결과와 COUNT(속성명)의 결과는 상이하다.
다. COUNT(*)의 처리 대상은 투플의 집합이고, 그 결과는 투플의 수이다.
라. COUNT(*)의 결과에 중복되는 투플은 포함된다.
보기

① 가, 나
② 가, 다
③ 나, 다
④ 다, 라

정답

정답 : ④번 (다, 라)

정답 신뢰도 : 매우 높음 (원본 이미지 Bold 확인 + COUNT 의미 검증)

핵심 키워드
  • COUNT(*) = 행(투플) 수(중복·NULL 포함)
  • COUNT(속성) = NULL 제외 값 수

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

결과 RS의 스키마가 R의 속성(A, B, C)만으로 구성되고, S와 (B, C)로 매칭되는 R의 투플만 남아 있다(a1·a2는 b1c1로 매칭, a4는 b2c3로 매칭, a3는 b1c2로 매칭 실패해 제외).

이처럼 "한쪽 릴레이션(R)의 속성만 유지하면서 다른 릴레이션(S)과 매칭되는 투플만 반환"하는 연산은 세미 조인(R ⋉ S)이다. 자연 조인(①)이라면 S의 속성(D)까지 결합되어 스키마가 (A,B,C,D)가 되므로 다르다. 따라서 정답은 ③번이다.

오답 분석

보기①

자연 조인은 S의 속성(D)까지 포함해 결과 스키마가 (A,B,C,D)가 된다. RS는 (A,B,C)뿐이므로 다르다.

보기②

우측 외부 조인은 S의 미매칭 투플까지 NULL 보충으로 포함하며 스키마도 다르다.

보기③

정답. 세미 조인은 R의 속성만 유지하고 S와 매칭되는 R 투플만 반환한다.

보기④

외부 합집합은 서로 다른 스키마를 합치는 연산으로 결과가 다르다.

관련 이론 정리

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

전역 깊이(가)는 디렉토리 엔트리 수로 결정된다. 디렉토리가 000~111의 8개(2³) 엔트리를 가지므로 전역 깊이 = 3이다.

지역 깊이(나)는 해당 버킷을 가리키는 디렉토리 엔트리 수와 관계된다. 어떤 버킷이 2개의 디렉토리 엔트리에 의해 공유되어 가리켜지면, 그 버킷의 지역 깊이 = 전역 깊이 − log₂(가리키는 엔트리 수) = 3 − log₂(2) = 3 − 1 = 2이다.

따라서 (가) 3, (나) 2로 정답은 ①번이다. (지역 깊이는 항상 전역 깊이 이하이며, 여러 엔트리가 한 버킷을 공유할수록 지역 깊이가 작다.)

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

"부양가족이 없는 사원"은 각 사원(EMPLOYEE)에 대해 그 사원(SSN)을 참조하는 부양가족(DEPENDENT.ESSN)이 존재하지 않는 경우다. 이를 표현하는 표준 패턴은 NOT EXISTS + 상관 조건(SSN=ESSN)이다.

WHERE NOT EXISTS (SELECT * FROM DEPENDENT WHERE SSN=ESSN)
- ㉮ = NOT EXISTS (존재하지 않음)
- ㉯ = * (EXISTS는 존재 여부만 보므로 SELECT * 사용)
- ㉰ = SSN=ESSN (사원과 부양가족의 상관 연결)

따라서 정답은 ①번이다. ②·③의 NOT IN은 상관 조건(SSN=ESSN)과 함께 쓰는 구조가 아니며, ④의 상관 조건(NAME=DEPNAME)은 사원-부양가족 연결이 아니다.

릴레이션 R을 릴레이션 R₁과 R₂로 분해할 때 바람직한 분해(decomposition)에 대한 설명으로 가장 적절하지 않은 것은? (단, R₁∩R₂는 R₁과 R₂의 공통 속성을 의미한다.)

보기

① 손실없는 분해(lossless decomposition)가 되어야 한다.
② 함수적 종속성(functional dependency) R₁∩R₂ → R₁ 과 R₁∩R₂ → R₂ 를 모두 만족하여야 한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

무손실(lossless-join) 분해의 조건은 공통 속성 R1∩R2가 R1 또는 R2 중 적어도 하나를 함수적으로 결정(즉 한쪽의 수퍼키가 됨)하는 것이다. 즉 (R1∩R2)→R1 또는 (R1∩R2)→R2 중 하나만 성립하면 된다.

②는 "R1∩R2→R1 과 R1∩R2→R2 를 모두 만족하여야 한다"고 했으나, 두 조건을 모두 만족할 필요는 없다(하나면 충분). 따라서 ②가 적절하지 않다. 정답은 ②번이다.

①(무손실 분해), ③(공통 속성이 한쪽 수퍼키 = 무손실 조건), ④(종속성 보존)는 바람직한 분해의 옳은 조건이다.

오답 분석

보기①

옳음. 바람직한 분해는 무손실이어야 한다.

보기②

정답(부적절). 무손실은 (R1∩R2)→R1 또는 →R2 중 하나면 충분하다. "모두 만족"은 과잉 조건이다.

보기③

옳음. 공통 속성이 R1의 수퍼키가 되면 무손실 조건을 만족한다.

보기④

옳음. 함수 종속성은 분해 후에도 가급적 보존되어야 한다(종속성 보존).

관련 이론 정리

바람직한 분해 조건:

조건 내용
무손실 (R1∩R2)→R1 또는 →R2(하나)
종속성 보존 원래 FD 보존
공통 속성 한쪽의 수퍼키
오해 "양쪽 모두 결정"은 불필요
실무 적용 사례

정규화 분해 감리 시 무손실(공통 속성이 한쪽 수퍼키)과 종속성 보존을 검증하되, 불필요하게 양쪽 결정을 요구하지 않도록 한다.

감리사 시험 출제 포인트

무손실 조건이 "또는"(하나)임을 이해했는지 묻는다. "모두 만족"으로 과잉 서술한 함정(②)을 골라낸다.

연관 규칙 생성을 위한 Apriori 알고리즘을 사용하여 {A,B,C,D}(간략히 ABCD라 함)가 빈발 항목집합(frequent itemset)임을 알았다. 그런데, ABCD에서 만든 규칙 BCD→A가 최소신뢰도 조건을 만족하지 않아 연관규칙이 될 수 없다면, 이 사실로부터 더 이상 고려할 필요가 없는 규칙들로만 짝지어진 것은?

② AB→CD, A→BCD

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

동일 빈발집합 ABCD에서 만든 규칙의 신뢰도 = supp(ABCD) / supp(선행부)이다. 선행부가 작아질수록(항목을 결론으로 옮길수록) 분모 supp(선행부)가 커져 신뢰도가 감소한다.

BCD→A(선행부 BCD)가 최소신뢰도를 만족하지 못하므로, 선행부가 BCD의 부분집합이면서 결론에 A를 포함하는 규칙은 신뢰도가 더 낮아 반드시 실패한다.
- BD→AC: 선행부 BD(⊂BCD), 결론 AC(⊇A) → supp(BD)≥supp(BCD)이므로 신뢰도 ≤ BCD→A → 실패(가지치기).
- D→ABC: 선행부 D(⊂BCD), 결론 ABC(⊇A) → supp(D)≥supp(BCD)이므로 신뢰도 ≤ BCD→A → 실패(가지치기).

따라서 더 이상 고려할 필요가 없는 규칙은 BD→AC와 D→ABC이며 정답은 ③번이다. (AB→CD, A→BCD 등 선행부가 BCD의 부분집합이 아니거나 결론에 A가 없는 규칙은 이 사실만으로 판단할 수 없다.)

오답 분석

보기①

BD→AC는 가지치기 대상이나, AB→CD는 선행부 AB가 BCD의 부분집합이 아니어서 이 사실만으로 실패를 단정할 수 없다.

보기②

AB→CD, A→BCD 모두 선행부가 BCD의 부분집합이 아니거나 판단 불가라 가지치기 대상이 아니다.

보기③

정답. BD→AC, D→ABC는 선행부⊆BCD, 결론⊇A로 신뢰도가 BCD→A 이하 → 확실히 실패.

보기④

A→BCD는 선행부 A가 BCD의 부분집합이 아니어서 이 사실만으로 판단할 수 없다.

관련 이론 정리

규칙 신뢰도 단조성:

원리 내용
신뢰도 supp(전체)/supp(선행부)
선행부 축소 분모↑ → 신뢰도↓
가지치기 실패 규칙의 선행부 부분집합·결론 확대 규칙도 실패
판단 불가 선행부가 실패 규칙 선행부의 부분집합이 아닐 때
실무 적용 사례

연관규칙 마이닝 감리 시 신뢰도 단조성으로 불필요한 규칙 후보를 조기 제거해 계산량을 줄이는 가지치기가 올바르게 동작하는지 검토한다.

감리사 시험 출제 포인트

"선행부를 줄이면(결론을 늘리면) 신뢰도가 감소한다"는 단조성으로, 실패 규칙(BCD→A)으로부터 가지치기 가능한 규칙을 식별한다.
