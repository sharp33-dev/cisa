정답 해설

투플 삽입 시 네 제약조건 모두 위배될 수 있다.
- 가(도메인 제약): 삽입 값이 속성의 도메인(자료형·범위)을 벗어나면 위배.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

약한 엔티티는 자체적으로 키를 형성할 충분한 속성이 없어, 소유(강한) 엔티티에 종속된다. 약한 엔티티를 유일하게 식별하려면 소유 엔티티의 기본키(식별자)와 약한 엔티티의 부분키(discriminator)를 결합한다. "강한 엔티티의 애트리뷰트들 중 임의의 하나와 결합"하는 것이 아니라, 반드시 소유 개체의 기본키를 사용해야 한다.

따라서 ③은 "임의의 하나와 결합"이라고 잘못 서술했으므로 옳지 않다. 정답은 ③번이다. ①(강한 개체=소유 개체), ②(자체 키 부족), ④(소유 개체 필요)는 모두 옳다.

오답 분석

보기①

옳음. 강한 엔티티가 약한 엔티티의 소유(식별) 엔티티가 된다.

보기②

옳음. 약한 엔티티는 키를 형성하기에 충분한 속성을 갖지 못한다.

보기③

정답(옳지 않음). 임의의 속성이 아니라 소유 엔티티의 기본키와 부분키를 결합해 식별한다.

보기④

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

차수(degree)는 결과의 열 수다. SELECT 절이 country_name, COUNT(city.lat) 두 열이므로 차수 = 2이다.

카디널리티는 결과 행 수다. INNER JOIN으로 도시가 있는 국가만 남으며, 국가별 도시 수는:
- 독일(Berlin)=1, 세르비아(Belgrade)=1, 크로아티아(Zagreb)=1, 미국(New York·Los Angeles)=2, 폴란드(Warsaw)=1.
- 스페인·러시아는 도시가 없어 조인에서 제외.

HAVING COUNT<=1 조건으로 도시가 1개인 국가만 남긴다 → 독일·세르비아·크로아티아·폴란드 = 4개(미국은 2개라 제외). 따라서 카디널리티 = 4.

결과는 차수 2, 카디널리티 4로 정답은 ③번이다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

회원은 로커가 필수(모든 회원이 로커 보유)이고, 로커는 회원이 선택적이다. 1:1 관계에서 외래키는 전체 참여(필수) 측에 두는 것이 유리하다. 즉 회원(MEMBER) 테이블에 LID를 외래키로 두고(가), 모든 회원이 로커를 가지므로 그 외래키를 NOT NULL로 정의(라)해 필수 참여를 강제한다.

만약 로커에 MID를 두면 회원이 없는 로커는 MID가 NULL이 되어 필수 참여를 강제하기 어렵다. 따라서 정답은 가·라인 ②번이다.

오답 분석

보기①

보기②

정답. MEMBER에 LID 외래키(가) + NOT NULL(라)로 회원의 필수 로커 배정을 강제.

보기③

나(LOCKER에 MID)는 회원 필수 참여 강제에 부적합하고 다(NULL)도 부적절하다.

보기④

나(LOCKER에 MID)는 로커가 회원 선택적이므로 MID가 NULL이어야 해 NOT NULL(라)과 모순된다.

3. 라: UNDO-list의 미완료 트랜잭션에 대해 UNDO 수행(역방향 취소).
4. 가: REDO-list의 커밋 트랜잭션에 대해 REDO 수행(정방향 재적용).

따라서 순서는 다 → 나 → 라 → 가로 정답은 ④번이다. (UNDO를 REDO보다 먼저 수행하는 것이 ARIES 등 표준 회복 절차의 특징이다.)

정답 해설

MAC의 대표 모델인 Bell-LaPadula(BLP)의 쓰기(갱신) 규칙(*-property, no write down)은 "사용자의 허가 등급이 객체의 비밀 등급보다 낮거나 같아야(즉 write up만 허용)" 쓰기가 가능하다. 이는 높은 등급 사용자가 낮은 등급 객체에 정보를 흘리는(write down) 것을 막기 위함이다.

③은 갱신 시 "허가 등급이 비밀 등급보다 같거나 높아야 한다"고 서술했는데, 이는 write down을 허용하는 것으로 BLP의 쓰기 규칙에 어긋난다. 따라서 ③이 옳지 않다. 정답은 ③번이다.

①(등급 기반 접근), ②(TS>S>C>U), ④(다단계 보안)는 옳다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

각 항목의 등장 거래 수: A={1,2,3,5}=4, B={1,3,4}=3, C={1,5,6}=3.

신뢰도 = support(X∪Y) / support(X):
- ① A→B: support(A,B)={1,3}=2 → 2/4 = 0.5
- ② A→C: support(A,C)={1,5}=2 → 2/4 = 0.5
- ③ B→A: support(A,B)=2 → 2/3 ≈ 0.667
- ④ B→C: support(B,C)={1}=1 → 1/3 ≈ 0.333

가장 큰 신뢰도는 B→A의 0.667이다. 따라서 정답은 ③번이다. 분모(선행 항목의 지지)가 작을수록(B=3) 같은 공동 지지(2)에 대해 신뢰도가 커진다.

오답 분석

보기①

A→B 신뢰도 = 2/4 = 0.5.

보기②

A→C 신뢰도 = 2/4 = 0.5.

보기③

정답. B→A 신뢰도 = 2/3 ≈ 0.667(최대).

보기④

B→C 신뢰도 = 1/3 ≈ 0.333.

장바구니 분석 감리 시 신뢰도가 선행 항목 지지에 의존함을 이해하고, 신뢰도·향상도를 함께 보아 의미 있는 규칙을 선별하는지 검토한다.

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

데이터 웨어하우스의 4대 특징은 주제지향(subject-oriented), 통합(integrated), 시계열성(time-variant), 비휘발성(nonvolatile)이다.

③은 "시간에 따라 변화하지 않는(time-invariant)"이라 서술하고 "DW는 현재 시점 데이터만 유지"한다고 했으나, 이는 정반대다. DW는 시계열성(time-variant)을 가져 과거 이력 데이터를 함께 유지하며(스냅샷 축적), 오히려 일반 운영 DB가 현재 시점 데이터 중심이다. 용어(time-invariant)와 설명 모두 틀렸으므로 ③이 옳지 않다. 정답은 ③번이다.

①(주제지향), ②(통합), ④(비휘발성)는 옳다.

오답 분석

보기①

옳음. DW는 의사결정 주제 중심으로 데이터를 구성(주제지향).

보기②

옳음. 여러 원천에서 추출·통합해 분석 지원(통합).

보기③

정답(옳지 않음). DW는 time-variant로 과거 이력을 유지한다. "time-invariant, 현재 데이터만 유지"는 틀림.

보기④

옳음. DW는 읽기 전용(비휘발성)으로 갱신이 드물다.

실무 적용 사례

정답 해설

TFIDF = TF × IDF이며, IDF = log(N / df)이다(N=전체 문서 수, df=용어가 등장한 문서 수).

용어 "this"는 d1과 d2 두 문서 모두에 등장하므로 df = 2, 전체 문서 N = 2이다. 따라서:

IDF("this") = log(N/df) = log(2/2) = log(1) = 0

IDF가 0이므로 TF 값과 무관하게 TFIDF = TF × 0 = 0이 된다(d1, d2 모두). 즉 모든 문서에 나타나는 흔한 용어는 변별력이 없어 TFIDF가 0으로 소멸한다. 정답은 ①번이다.

정답 해설

혼동행렬: TP=45, FN=20, FP=5, TN=30.

  • ① 정밀도 = TP/(TP+FP) = 45/(45+5) = 45/50 → 옳음.
  • ② 재현율 = TP/(TP+FN) = 45/(45+20) = 45/65 → 옳음.
  • ③ 특이도 = TN/(TN+FP) = 30/(30+5) = 30/35이어야 하는데, 보기는 20/50으로 잘못됨 → 옳지 않음.
  • ④ 정확도 = (TP+TN)/전체 = (45+30)/100 = 75/100 → 옳음.

따라서 값이 옳지 않은 것은 ③번 특이도이다(30/35가 정답인데 20/50으로 표기).

보기

① 이름부여 투명성(naming transparency)은 분산 데이터베이스가 어떻게 설계되는지와 어느 사이트에서 트랜잭션이 실행되는지를 사용자에게 가려주는 것을 의미한다.
② 분산 투명성(distribution transparency) 또는 네트워크 투명성(network transparency)은 사용자에게 네트워크의 세부 사항과 분산 시스템 내에서 데이터 저장 장소의 위치를 가려주는 것을 의미한다.
③ 위치 투명성(location transparency)은 어떤 작업을 수행하기 위해 사용된 명령은 데이터의 위치와 명령이 입력된 시스템의 위치와 무관해야 함을 의미한다.
④ 중복 투명성(replication transparency)은 높은 가용성, 성능, 신뢰도를 위해 데이터 사본들이 여러 사이트에 중복될 수 있음을 의미하며, 중복된 사본의 존재를 사용자에게 가려준다.

정답 해설

이름부여 투명성(naming transparency)은 각 데이터 객체가 유일한 이름을 가지며, 사용자가 객체의 물리적 위치나 사이트를 몰라도 이름만으로 접근할 수 있음을 의미한다.

①은 이름부여 투명성을 "분산 DB가 어떻게 설계되는지와 어느 사이트에서 트랜잭션이 실행되는지를 가려주는 것"이라고 서술했는데, 이는 이름부여 투명성의 정의가 아니다(설계·실행 사이트 은닉은 다른 개념/부정확한 설명). 따라서 ①이 적절하지 않다. 정답은 ①번이다.

②(분산/네트워크 투명성), ③(위치 투명성), ④(중복 투명성)는 각각 정의와 대체로 일치한다.

오답 분석

보기①

정답(부적절). 이름부여 투명성은 "이름만으로 접근(위치 무관)"이지, "설계·트랜잭션 실행 사이트 은닉"이 아니다.

보기②

옳음. 분산(네트워크) 투명성은 네트워크 세부·저장 위치를 가려준다.

보기③

옳음. 위치 투명성은 명령이 데이터·입력 시스템 위치와 무관하게 동작함을 의미한다.

보기④

옳음. 중복 투명성은 사본의 존재를 사용자에게 가려준다.
