- 비즈니스 룰 → 기능적 요구사항에 제약/영향

public static int search(List<Integer> list, int k) {
    int left = 0, right = list.size()-1, result = -1;
    while (left <= right) {
        int mid = left + ((right - left) / 2);
        if (list.get(mid) > k) {
            right = mid - 1;
        } else if (list.get(mid) == k) {
            result = mid;
            right = mid - 1;
        } else {
            left = mid + 1;
        }
    }
    return result;
}
// 입력: {1,2,3,4}, k=3

영역 2 · 정답 해설 / 오답 분석 / 이론
정답 해설

MC/DC 조건: 각 개별 조건이 독립적으로 전체 결정에 영향을 줘야 한다.

조건: (x > 0) AND (y <= 0)

C1(x>0) 독립성 검증 - C2=T 고정, C1 변화:
- TC1: C1=T, C2=T → 결과 T
- TC3: C1=F, C2=T → 결과 F (AND이므로 F)
- → C1이 T→F가 될 때 결과도 T→F: C1의 독립 영향 확인 ✓

C2(y<=0) 독립성 검증 - C1=T 고정, C2 변화:
- TC1: C1=T, C2=T → 결과 T
- TC2: C1=T, C2=F → 결과 F (AND에서 C2=F이면 F)
- → C2가 T→F가 될 때 결과도 T→F: C2의 독립 영향 확인 ✓

필요한 최소 TC: TC1, TC2, TC3 (3개)

TC4(F, F): C1=F이면 AND 단락 평가(Short-circuit)로 C2를 평가하지 않음. C2의 독립적 영향을 확인하려면 C1=T 상태에서 C2를 변화시켜야 하므로, TC4는 MC/DC 달성에 불필요하다.

관련 이론 정리

MC/DC for AND 조건:

조건 필요 TC 쌍 이유
C1 독립 TC1(T,T)↔TC3(F,T) C2=T 고정, C1만 변화
C2 독립 TC1(T,T)↔TC2(T,F) C1=T 고정, C2만 변화

최소 TC = {TC1, TC2, TC3} → TC4 불필요

정답 해설

시퀀스 다이어그램 분석:
- c2s: C2라는 표기는 C2 타입의 컬렉션을 의미
- loop [for each c2 in c2s]: C2 객체들의 컬렉션을 순회
- 즉, C1은 여러 개의 C2 객체를 보유

클래스 다이어그램 문제:
- C1-C2 연관관계에 다중성이 없음
- 시퀀스 다이어그램에서 C1이 C2의 컬렉션(c2s)을 가지므로, 클래스 다이어그램에서 C2 측에 * 다중성이 명시되어야 함
- * = 0 이상의 여러 C2 인스턴스

→ ③번이 정답

- 공유 컴포넌트 (Reusable Components)
- 도메인 특화 (Domain-Specific)
- 고객별 변형 가능 (Variability Management)

보기①(아키텍처 패턴)

정답 해설

CC(순환 복잡도) 계산:

시퀀스 다이어그램의 결합 프래그먼트(Combined Fragment):
- loop → 반복 결정 노드 1개
- break → 조건 결정 노드 1개
- opt → 선택 결정 노드 1개

결정 노드 수 = 3개
CC = 3 + 1 = 4

CBO(결합도) 계산:

CBO = Class1이 직접 사용하는(메시지를 보내는) 고유 클래스 수 (Class1 제외, 단방향)

Class1이 메시지를 보내는 클래스:
- O21: Class2 → Class2
- O22: Class2 → Class2 (이미 카운트)
- Class3 (break에서 m30 호출)

고유 클래스: Class2, Class3 = 2개
CBO = 2

→ CC = 4, CBO = 2 = ④번

관련 이론 정리

CBO 계산 규칙:
- Class1이 사용하는 클래스 수 (단방향, 고유 클래스)
- 동일 클래스의 다른 인스턴스는 1개로 카운트
- Class1을 사용하는 클래스는 포함하지 않음(문제 조건)
